# 10.4 Filutdelning i lokala nätverk (SMB och NFS)

När vi arbetar i ett nätverk räcker det inte alltid med att skicka enstaka filer via SFTP (se 10.1). Ofta behöver flera användare ha tillgång till samma mappar samtidigt, eller så vill man att en server ska fungera som ett gemensamt arkiv där filer kan öppnas och sparas direkt utan att först laddas ner. För detta används protokoll för filutdelning. De två dominerande språken för detta är SMB (för Windows) och NFS (för Linux/Unix).

### 10.4.1 SMB – Standarden för Windows

SMB (Server Message Block) är det protokoll som Windows använder för att dela ut filer och skrivare. Du känner säkert igen det som "delade mappar" i Utforskaren. SMB kommunicerar idag främst över port 445.

Utöver fildelning är SMB ett kraftfullt protokoll för att dela kringutrustning i nätverket. Det stödjer delning av enheter såsom skrivare och scanners. Genom SMB kan en central server göra en fysiskt ansluten scanner eller skrivare tillgänglig för alla datorer i det lokala nätverket, vilket gör att användarna kan hantera dokument som om enheten satt kopplad direkt i deras egen dator.

Tekniken har utvecklats mycket över tid. Den tidiga versionen (SMBv1, även kallad CIFS) skapades när säkerhet inte prioriterades. I dessa gamla versioner skickades mycket av informationen på ett sätt som gjorde det enkelt för någon med en nätverkssniffer (som Wireshark) att se vad som pågick. Idag används SMBv3, som har inbyggd kryptering. Det innebär att även om någon avlyssnar nätverket, kan de inte läsa innehållet i filerna som öppnas eller sparas.

En viktig detalj med SMB är att det hanterar både autentisering (att kontrollera vem du är med användarnamn och lösenord) och behörigheter (vad du får göra med filen). Det är detta som gör att en lärare kan ha en mapp som eleverna bara får läsa ifrån, medan läraren själv kan ändra i filerna.

### 10.4.2 NFS – Favoriten i Linux-världen

NFS (Network File System) är motsvarigheten i Linux- och Unix-miljöer. Där SMB ofta används för att människor ska kunna dela kontorsdokument, används NFS ofta för att servrar ska kunna dela systemfiler eller databaser med varandra. NFS kommunicerar vanligtvis över port 2049.

NFS är byggt för att vara snabbt och effektivt. När en Linux-dator ansluter till en NFS-utdelning så "monteras" den i filsystemet. Det innebär att den delade mappen ser ut och fungerar exakt som vilken lokal mapp som helst på hårddisken.

Säkerheten i äldre versioner av NFS byggde ofta på att man litade på den anslutande datorns IP-adress. Om din dator hade "rätt" IP-adress fick du tillgång till filerna. Detta är en stor säkerhetsrisk, eftersom en angripare kan förfalska sin IP-adress (IP-spoofing) för att lura servern. Moderna versioner av NFS kan dock använda starkare skydd som kräver att både användaren och datorn bevisar sin identitet med digitala nycklar.

### 10.4.3 Att blanda världar: Samba

Eftersom Windows pratar SMB och Linux pratar NFS kan de normalt sett inte förstå varandras filutdelningar direkt. För att lösa detta finns en programvara som heter Samba. Samba gör det möjligt för en Linux-server att "låtsas" vara en Windows-server och prata SMB.

Tack vare Samba kan en Linux-maskin fungera som en komplett fil- och utskriftshub för ett kontor med Windows-klienter. Det är Samba som gör att du kan sitta vid en Windows-dator och spara filer på en server som kör Linux, eller använda en nätverksskrivare kopplad till en Linux-dator, utan att du märker någon skillnad.

### 10.4.4 CUPS – Utskriftshantering i Linux

Medan SMB ofta hanterar utskrifter i Windows, används CUPS (Common Unix Printing System) som standard i Linux och macOS. CUPS är en modulär utskriftstjänst som gör att en dator kan fungera som en utskriftsserver.

CUPS använder protokollet IPP (Internet Printing Protocol), vanligtvis över port 631, för att hantera utskriftsköer och kommunicera med skrivare. En stor fördel med CUPS är dess webbaserade gränssnitt som gör det enkelt för en tekniker att administrera skrivare, se utskriftshistorik och styra behörigheter via en vanlig webbläsare. Genom att kombinera CUPS med Samba kan en Linux-server ta emot utskriftsjobb från Windows-klienter (via SMB) och sedan skicka dem vidare till den fysiska skrivaren.
