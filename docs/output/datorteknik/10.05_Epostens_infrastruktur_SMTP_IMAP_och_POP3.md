# 10.5 E-postens infrastruktur (SMTP, IMAP och POP3)

Trots uppkomsten av snabbmeddelanden och samarbetsplattformar är e-post fortfarande ryggraden i affärskommunikation. För att ett e-postmeddelande ska färdas från din dator, genom internet och landa i mottagarens inkorg, krävs ett samspel mellan olika protokoll som har specifika ansvarsområden.

### 10.5.1 SMTP – Den digitala brevbäraren

SMTP (Simple Mail Transfer Protocol) är det protokoll som används för att skicka e-post. När du trycker på "Skicka" i ditt e-postprogram pratar programmet med en SMTP-server.

SMTP fungerar som en brevbärare som tar emot ditt brev och ser till att det transporteras till mottagarens postkontor (server). Traditionellt användes port 25 för detta, men eftersom den porten saknade kryptering och ofta missbrukades för skräppost (spam), använder moderna och säkra anslutningar idag oftast port 587 eller port 465. Precis som med HTTP (se 10.2) skickades SMTP ursprungligen i klartext, vilket innebar att vem som helst som avlyssnade nätverket med verktyg som Wireshark kunde läsa både meddelandet och inloggningsuppgifterna. Idag används nästan alltid kryptering för att skydda innehållet under transporten.

### 10.5.2 POP3 – Hämta och radera (Den gamla skolan)

När brevet väl har kommit fram till mottagarens server måste mottagaren kunna hämta det. POP3 (Post Office Protocol version 3) är ett äldre protokoll för detta ändamål.

POP3 fungerar ungefär som ett fysiskt postfack: du går till facket, tömmer det och tar med dig breven hem. När ditt e-postprogram hämtar brev via POP3 laddas de ner till din hårddisk och raderas vanligtvis från servern. Detta fungerade bra förr när man bara läste e-post på en enda dator, men i dagens värld där vi vill se samma mejl i mobilen, på surfplattan och på datorn är POP3 problematiskt, eftersom mejlet bara finns kvar på den enhet som först hann hämta det. POP3 använder normalt port 110 (okrypterat) eller port 995 (säkert).

### 10.5.3 IMAP – Synkronisering (Den moderna standarden)

IMAP (Internet Message Access Protocol) är det protokoll de flesta använder idag. Till skillnad från POP3 låter IMAP mejlen ligga kvar på servern. Ditt e-postprogram "speglar" bara det som finns på servern.

Detta innebär att om du läser ett mejl i mobilen eller flyttar det till en mapp, så ser du exakt samma ändring när du senare öppnar din dator. Allt är synkroniserat. IMAP är mer tekniskt avancerat och kräver mer lagringsutrymme på servern, men det är en förutsättning för ett modernt digitalt liv. IMAP använder normalt port 143 (okrypterat) eller port 993 (säkert).

### 10.5.4 E-post och säkerhet

E-post är historiskt sett ett osäkert medium. Utöver risken för avlyssning (som vi nu motverkar med kryptering på portarna ovan) finns problemet med att vem som helst kan påstå sig vara vem som helst i ett mejl. Detta kallas för spoofing (vi fördjupar oss i olika typer av spoofing-attacker i kapitel 11.2).

För att motverka detta använder servrar idag tekniker som kontrollerar att mejlet faktiskt kommer från den server det påstår sig komma ifrån. Som tekniker är det viktigt att känna till att krypteringen (SSL/TLS) bara skyddar mejlet medan det färdas mellan din dator och servern. När mejlet väl ligger lagrat på en server är det upp till serverns administratör att se till att ingen obehörig kan läsa det.
