# 13.8 Kriskommunikation och lagstadgad rapportering

När en allvarlig säkerhetsincident inträffar är det inte bara servrar och nätverk som sätts under press, utan även organisationens rykte och juridiska ansvar. Som vi såg i kapitel 12.2, är kommunikationsansvariga och juridiska experter centrala medlemmar i ett Incident Response Team (IRT). Deras uppgift är att se till att rätt information når rätt personer i rätt tid. Om tekniken fallerar är det kommunikationen som avgör om kundernas och allmänhetens förtroende kan räddas eller om incidenten leder till en förtroendekris.

### 13.8.1 Intern och extern kommunikation

Det första steget i kriskommunikation handlar om att ta kontroll över narrativet. Internt måste anställda snabbt få veta vad de ska göra – och vad de absolut inte ska göra. Det kan handla om instruktioner att inte logga in i vissa system eller att inte uttala sig i sociala medier. Ryktesspridning vid kaffemaskinen kan vara lika skadlig som själva viruset.

Externt är balansen svårare. Att vara för öppen för tidigt kan ge angriparen ledtrådar om att de är upptäckta, medan att mörka en incident kan leda till enorma böter och rykten om inkompetens. Här används ofta de mallar som finns i organisationens Incident Response Plan (IRP) för att snabbt få ut saklig information till kunder och samarbetspartners.

### 13.8.2 Den kritiska 72-timmarsregeln (GDPR)

Som vi diskuterade i kapitel 11.10 (Lagar och etik), ställer GDPR extremt hårda krav på rapportering. Om incidenten innebär att personuppgifter (PII) kan ha kommit på avvägar, har organisationen en lagstadgad skyldighet att anmäla detta till Integritetsskyddsmyndigheten (IMY).

Denna anmälan måste ske senast 72 timmar efter att incidenten upptäckts. Under denna korta tid måste IRT-teamet ha hunnit identifiera vad som hänt, vilka som drabbats och vilka åtgärder som vidtagits för att begränsa skadan. Om man missar denna deadline riskerar företaget administrativa sanktionsavgifter som kan uppgå till miljonbelopp. Utöver IMY kan det även krävas rapportering till Polisen om händelsen bryter mot Brottsbalkens kapitel om dataintrång (se kapitel 11.10.1).

### 13.8.3 Samverkan med nationella organ

Sverige har specialiserade myndigheter som stöttar vid it-incidenter. CERT-SE, som drivs av MSB, är Sveriges nationella CSIRT (Computer Security Incident Response Team). De fungerar som en länk mellan organisationer och kan varna andra om de ser att en specifik typ av attack, som exempelvis Log4Shell (se kapitel 11.12.5), sprider sig i landet. Att rapportera till CERT-SE handlar inte bara om lagkrav, utan om att bidra till det kollektiva försvaret av den svenska digitala infrastrukturen.
