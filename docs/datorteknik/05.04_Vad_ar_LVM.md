# 5.4 Vad är LVM?

LVM står för Logical Volume Manager. Istället för att direkt skapa partitioner på en fysisk disk, lägger LVM till ett extra lager.

LVM består av:

Fysiska volymer (PV) = t.ex. /dev/sda1

Volymgrupper (VG) = flera PV bildar en VG

Logiska volymer (LV) = själva partitionerna man monterar

När du installerar ett operativsystem för första gången är det lätt att tänka på hårddisken som en fast låda. Om lådan är 500 GB stor, så är den det. Men i servervärlden räcker det sällan med fasta storlekar. Tänk om din databas växer snabbare än förväntat, eller om du plötsligt behöver mer plats för användarnas filer? Det är här LVM (Logical Volume Management) kommer in i bilden.

Ett enkelt sätt att förklara LVM är att tänka på ett LEGO-hus. I stället för att vara låst vid en viss storlek kan du bygga om huset (disken) precis som du vill utan att behöva riva allt och börja om. Med LVM kan du ändra storlek på dina "rum" (volymer), lägga till mer lagring och flytta runt data medan systemet är igång. Behöver du fler rum eller ett större hus? Då lägger du bara till fler bitar utan större ansträngning.

Detta skiljer sig markant från traditionell RAID-hantering. Om du skulle behöva utöka en vanlig RAID med fler diskar krävs ofta att strömmen bryts och nya diskar monteras fysiskt. Därefter måste hela RAID-matrisen "räknas om" eftersom all information nu ska spridas över de nya diskarna. Under denna tid kan prestandan bli lidande eller så går enheten inte att använda alls. Det är en process som är enormt tidskrävande och kan ta allt från timmar till dagar. LVM ger oss istället en mjukvarubaserad flexibilitet som gör att vi kan hantera lagring nästan som om den vore flytande.

LVM används ofta tillsammans med RAID eller ZFS för att ge ännu större flexibilitet och kontroll över lagringen.

Fördelar:

Flexibilitet

Kan ändra storlek på diskar live

Snapshot-funktioner

Nackdelar:

Lite mer komplext

Mjukvarubaserat

Windowsstöd saknas. Windows kan installeras på en LV men kan inte hantera LVM

### 5.4.1 Exempel från verkligheten

Ubuntu Server: Vid installation kan man välja LVM direkt. Kombinera med RAID via mdadm eller ZFS.

Proxmox: Använder ofta ZFS som grund, för snapshots, replikering och säker lagring.

NAS-lösningar (t.ex. TrueNAS): Använder ZFS som standard.
