# 11.6 Vanliga angrepp mot klienter och system

För att kunna bygga ett effektivt försvar måste vi förstå hur ett angrepp ser ut i praktiken. En angripare letar sällan efter den svåraste vägen in; de letar efter den mest effektiva. I en Enterprise-miljö pratar vi ofta om Attack Vectors, vilket är den väg eller det sätt som skadlig kod eller en angripare använder för att få tillträde till ett system.

### Malware – Skadlig programvara

Malware är ett samlingsnamn för all mjukvara med syfte att skada, störa eller få obehörig åtkomst. Det är viktigt att skilja på typerna för att veta hur de stoppas.

Virus: Kräver mänsklig interaktion för att sprida sig, till exempel att någon kör en infekterad fil.

Worms: Självloppande program som sprider sig automatiskt över nätverket genom att utnyttja sårbarheter i operativsystem eller tjänster. En mask kan sänka ett helt företagsnätverk på några minuter.

Trojans: Utger sig för att vara ett legitimt program men döljer en skadlig funktion. Används ofta för att skapa en Backdoor så att angriparen kan återvända senare.

Ransomware: Den idag mest lönsamma formen av malware. Den krypterar offrets filer och kräver en lösensumma. I professionella miljöer siktar angreppet ofta på att även förstöra backuper.

Spyware och Keyloggers: Program som döljer sig i systemet för att stjäla information genom att registrera vad användaren gör eller skriver.

### Avancerade avlyssnings- och manipulationsangrepp

Dessa angrepp fokuserar på att kontrollera eller påverka kommunikationsflödet.

Man-in-the-Middle (MitM): Angriparen placerar sig logiskt mellan två parter. Detta kan ske genom tekniker som ARP Spoofing (i lokala nätverk) eller DNS Poisoning. Angriparen kan här både läsa och modifiera trafiken i realtid.

Man-on-the-Side (MotS): Till skillnad från MitM kontrollerar angriparen inte hela länken, utan sitter på sidan av och "ser" trafiken. Angriparen injicerar sedan ett eget svar som når offret snabbare än det legitima svaret (ett så kallat race condition). Detta kräver ofta kontroll över nätverksinfrastruktur på hög nivå.

Man-in-the-Browser (MitB): Här infekteras användarens webbläsare av en trojan. Angreppet sker "inuti" krypteringen; mjukvaran modifierar webbsidor eller transaktioner efter att de dekrypterats av webbläsaren men innan de visas för användaren. Vid en banköverföring kan angriparen ändra mottagarnummer och belopp i bakgrunden, medan användaren fortfarande ser de korrekta uppgifterna på skärmen.

### Fysiska angrepp

Evil Maid Attack: Detta är ett av de svåraste angreppen att skydda sig mot eftersom det kräver fysisk tillgång till hårdvaran. Namnet kommer från scenariot där en "ond städerska" får tillgång till en bärbar dator som lämnats kvar på ett hotellrum. Angriparen kan installera en fysisk keylogger eller modifiera datorns Bootloader för att stjäla krypteringsnycklar nästa gång användaren startar datorn. Även om hårddisken är krypterad kan en Evil Maid-attack kompromettera systemet innan dekrypteringen ens har skett.

### Angrepp mot applikationer och data

SQL Injection (SQLi): Angriparen skickar skadlig databaskod via inmatningsfält. Om mjukvaran saknar korrekt Input Validation kan angriparen "lura" databasen att lämna ut all information eller radera tabeller.

Cross-Site Scripting (XSS): Angriparen injicerar skadliga skript i en webbsida som sedan körs i den vanliga användarens webbläsare för att stjäla sessions-cookies.

### Exploatering och sårbarheter

När ett säkerhetshål upptäcks kallas det för en Exploit. Om sårbarheten är helt ny och det ännu inte finns någon Patch tillgänglig, kallas det för en Zero-day. Professionell IT-drift kräver effektiv Patch Management för att minimera tiden systemen är sårbara.
