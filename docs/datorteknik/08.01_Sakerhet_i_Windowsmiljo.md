# 8.1 Säkerhet i Windowsmiljö

Syftet med detta kapitel är att vara en fördjupning av de särkerhetsfunktioner som finns i Windows, och hur de används. Många av verktygen nämns i kapitlet innan, men här går vi in mer på djupet av dem

### 8.1.1 Grundläggande säkerhetsprinciper och Windows Security

Säkerhet i en Windowsmiljö handlar inte bara om att installera ett antivirusprogram. Eftersom Windows är världens mest använda operativsystem för arbetsstationer är det också det främsta målet för skadlig kod och nätfiske. För att skydda ett Windows-system krävs en kombination av rätt konfiguration, modern hårdvara och en förståelse för grundläggande säkerhetsprinciper.

#### Minsta möjliga behörighet (Least Privilege)

Oavsett vilket operativsystem man använder är principen om minsta möjliga behörighet fundamental i Windows. Historiskt sett kördes Windows-användare ofta som administratörer hela tiden, vilket innebar att om ett virus infekterade datorn fick viruset omedelbart full kontroll. I moderna Windows-versioner används User Account Control (UAC) för att separera vanliga användarrättigheter från administrativa uppgifter, vilket tvingar användaren att aktivt godkänna ändringar som påverkar systemet.

#### Defense in Depth – Lager på lager

Windows är uppbyggt för att erbjuda säkerhet i flera lager. Om en angripare tar sig förbi ett lager (t.ex. lurar användaren att ladda ner en fil), ska nästa lager (t.ex. realtidsskanning eller brandvägg) stoppa attacken.

Exempel på lager i Windows:

Hårdvarulagret: TPM och Secure Boot.

Operativsystemslagret: Kärnisolering och Windows Defender.

Nätverkslagret: Windows Firewall och SmartScreen.

Användarlagret: Multi-faktor autentisering (MFA) och Windows Hello.

#### Windows Security – Systemets kontrollcenter

Windows Security (tidigare Windows Security Center) är den centrala platsen där du får en översikt över dessa skyddslager. Det är här administratören snabbt kan se om något lager är inaktiverat eller kräver åtgärd. Här samlas de viktigaste funktionerna:

Virus- och hotskydd: Hanterar Microsoft Defender och realtidsskanning.

Brandvägg och nätverksskydd: Övervakar in- och utgående trafik.

App- och webbläsarkontroll: Använder SmartScreen för att blockera misstänkta filer och webbplatser.

Enhetssäkerhet: Ger tillgång till avancerade funktioner som kärnisolering.

Kontoskydd: Hanterar inloggningsalternativ och MFA.

#### Hårdvarubaserad säkerhet: TPM och Secure Boot

Modern Windows-säkerhet bygger på att mjukvaran samarbetar med hårdvaran för att skapa en säker grund.

TPM (Trusted Platform Module): En dedikerad säkerhetsmodul (chip) som lagrar krypteringsnycklar, lösenord och certifikat. Från och med Windows 11 är TPM 2.0 ett krav. Det används bland annat av BitLocker för att säkerställa att hårddisken inte kan läsas om den flyttas till en annan dator.

Secure Boot: En funktion i UEFI (moderna motsvarigheten till BIOS) som kontrollerar att endast kod med en giltig digital signatur får köras vid uppstart. Detta förhindrar rootkits och boot-attacker från att infektera datorn innan operativsystemet ens har laddats.

Bra att veta: Om du installerar ett tredjeparts antivirusprogram (t.ex. Norton eller Bitdefender), inaktiveras ofta de delar av Microsoft Defender som sköter realtidsskanning för att undvika konflikter, men Windows Security fortsätter att fungera som en instrumentpanel för att visa den övergripande hälsan.

### 8.1.2 UAC och Integrity Levels

User Account Control (UAC) är en säkerhetsfunktion som kontrollerar vem som får göra vad i systemet – särskilt när det gäller att ändra inställningar som påverkar hela datorn. För en IT-tekniker är UAC inte bara en "varningsruta", utan en sofistikerad kontrollmekanism som bygger på arkitekturen för Integrity Levels.

#### Varför finns UAC?

Innan UAC introducerades (t.ex. i Windows XP) kördes de flesta användare som administratörer hela tiden. Det innebar att om en användare råkade starta ett virus, fick viruset omedelbart fullständiga rättigheter att ändra systemfiler, installera bakdörrar och förstöra operativsystemet.

Med UAC körs alla program som "vanliga användare" som standard, även om du är inloggad som administratör. Det fungerar som en digital säkerhetsvakt vid dörren till systemets hjärta.

#### Den tekniska bakgrunden: Dual Tokens och Integrity Levels

När en administratör loggar in i moderna Windows skapas två separata tokens (åtkomstnycklar):

Standard User Token: Används för nästan allt du gör (surfa, skriva dokument).

Administrator Token: Innehåller alla rättigheter men ligger "vilande" tills det behövs.

För att bestämma vilket program som får prata med vilket, använder Windows Integrity Levels (Integritetsnivåer). Det förhindrar att ett program med låg säkerhet (t.ex. en webbläsare) skickar kommandon till ett program med hög säkerhet (t.ex. ett administrativt verktyg).

Bra att veta: En process på en lägre nivå kan aldrig "skriva" till en process på en högre nivå. Det kallas UIPI (User Interface Privilege Isolation).

#### UAC-nivåer i Windows 10/11

Du kan själv justera hur känslig säkerhetsvakten ska vara via Kontrollpanelen:

Alltid meddela: Den säkraste nivån. Frågar även om du själv gör ändringar i Windows-inställningar. Skrivbordet tonas ner (Secure Desktop) för att förhindra att skadlig kod "klickar" på knappen åt dig.

Meddela endast vid appar (Standard): Windows frågar bara när program försöker göra ändringar, inte när du själv ändrar inställningar.

Meddela endast vid appar (tona inte ner): Samma som ovan, men utan att frysa resten av skrivbordet. Detta är osäkrare då vissa typer av skadlig kod kan simulera musklick.

Aldrig meddela: UAC är i praktiken avstängt. Rekommenderas ej, då det tar bort skyddet helt och hållet.

### 8.1.3 Microsoft Defender & Ransomware-skydd

Microsoft Defender Antivirus (tidigare Windows Defender) har genomgått en enorm utveckling. Från att ha varit ett grundläggande komplement till tredjepartsprogram har det idag blivit en av de mest respekterade säkerhetslösningarna på marknaden, ofta med toppresultat i oberoende tester som AV-Test och AV-Comparatives.

Som tekniker räcker det inte med att veta att det "finns där" – du behöver förstå de avancerade funktionerna som gör det till ett proaktivt skydd.

#### Avancerade skyddsfunktioner

Utöver den klassiska sökningen efter kända virusfiler innehåller Defender flera moderna tekniker för att stoppa attacker innan de ens hunnit starta:

Tamper Protection (Manipulationsskydd): Detta är en av de viktigaste funktionerna för en administratör. Den hindrar skadliga appar (eller angripare som lyckats få administratörsrättigheter) från att ändra viktiga inställningar i Defender, som att stänga av realtidsskyddet eller molnbaserat skydd.

Molnbaserat skydd (Cloud-delivered Protection): När Defender stöter på en misstänkt fil som inte finns i den lokala databasen, skickas metadata till Microsofts moln. Där analyseras filen med hjälp av AI och maskininlärning. Om den bedöms som skadlig spärras den globalt inom några sekunder.

Skydd mot potentiellt oönskade appar (PUA): Blockerar mjukvara som inte nödvändigtvis är virus, men som kan påverka prestandan eller integriteten negativt, till exempel adware, webbläsarkapare eller "systemoptimerare".

#### Ransomware-skydd: Controlled Folder Access

Ett av de största hoten mot moderna Windows-miljöer är ransomware. Microsoft har svarat på detta med funktionen Kontrollerad mappåtkomst (Controlled Folder Access).

Denna funktion fungerar som en vakt vid dörren till dina viktigaste mappar (t.ex. Dokument, Bilder och Skrivbord). Endast program som Windows betraktar som säkra får skriva till eller ändra filer i dessa mappar. Om ett okänt program (t.ex. ett ransomware-skript) försöker kryptera dina filer, blockeras handlingen omedelbart och du får en notifiering.

#### Så fungerar Kontrollerad mappåtkomst (Controlled Folder Access)

Tekniken bygger på en tillåtelse-lista (allow-list). Istället för att Windows försöker gissa vilka program som är elaka (svartlistning), vänder man på steken: Windows blockerar alla program från att skriva i dina viktiga mappar, förutom de som Microsoft vet är säkra eller de som du själv har godkänt.

Om ett okänt program försöker ändra en fil i en skyddad mapp, stoppas processen omedelbart. Windows genererar då en loggpost i Loggboken (Event Viewer) och skickar en notifiering till användaren.

#### Här hittar du funktionen

Eftersom detta är en del av Ransomware-skyddet ligger det djupt inne i säkerhetsinställningarna:

Öppna Windows-säkerhet (sök i Start-menyn).

Klicka på Virus- och hotskydd.

Skrolla ner till botten och klicka på Hantera ransomware-skydd.

Här hittar du reglaget för Kontrollerad mappåtkomst.

#### Aktivering och konfiguration (Steg-för-steg)

Som tekniker behöver du veta hur du finjusterar skyddet så att det inte stör användarens arbete:

Slå på skyddet: Dra reglaget till På. (Kräver administratörsbehörighet via UAC).

Skyddade mappar: Som standard skyddas systemmappar (Dokument, Bilder, Skrivbord m.m.). Du kan klicka på Skyddade mappar för att lägga till egna, till exempel en mapp med viktiga projektfiler på en annan hårddisk.

Tillåt en app: Detta är den viktigaste fliken. Om en användare ringer och säger "Mitt ritprogram kan inte spara filer!", går du hit. Genom att klicka på Tillåt en app via kontrollerad mappåtkomst kan du manuellt lägga till programvarans .exe-fil i undantagslistan.

Bra att veta: I en företagsmiljö aktiverar man oftast inte detta manuellt på varje dator. Istället använder man Gruppolicy (GPO) eller Microsoft Intune för att rulla ut inställningen och undantagslistan till alla datorer samtidigt. Man börjar ofta i "Audit Mode" (granskningsläge) för att se vilka program som skulle ha blockerats innan man slår på det "på riktigt".

#### Microsoft Defender for Endpoint (EDR)

För företag räcker det inte med att bara ha ett antivirus som reagerar när en fil redan har landat på hårddisken. Microsoft Defender for Endpoint är en molnbaserad EDR-plattform (Endpoint Detection and Response) som fungerar som systemets "svarta låda".

Istället för att bara leta efter kända virusfiler, samlar EDR in telemetri (data om beteenden) från alla datorer i nätverket. Den ser om en användare i Stockholm plötsligt kör ett märkligt PowerShell-kommando, samtidigt som en dator i Malmö försöker ansluta till en misstänkt IP-adress i utlandet. Genom att lägga ihop dessa pusselbitar kan molntjänsten upptäcka avancerade attacker som försöker "flyga under radarn".

#### Hur det fungerar i praktiken

EDR-plattformen analyserar ständigt tre huvudområden:

Beteendeanalys: Ser om ett program gör något det aldrig gjort förut, som att försöka läsa av lösenord ur systemminnet.

Sårbarhetshantering: Rapporterar automatiskt om datorerna har föråldrad mjukvara (t.ex. en gammal version av Chrome) som har kända säkerhetshål.

Automatisering (AIR): Systemet kan själv starta en undersökning, isolera filer och återställa ändringar utan att teknikern behöver göra något manuellt.

#### Var hittar jag det?

Till skillnad från det vanliga Defender-gränssnittet, som du hittar lokalt på datorn, hanteras Defender for Endpoint via en webbportal: Microsoft Defender-portalen (security.microsoft.com).

Här kan du som administratör:

Isolera enheter: Med ett klick koppla bort en infekterad dator från nätverket (men behålla kontakten med molnet) så att smittan inte sprids.

Göra en "Live Response": Gå in i en terminal på distans för att samla in filer eller köra skript för att rensa datorn.

Se tidslinjer: Se exakt vad som hände på datorn sekund för sekund innan larmet gick.

#### Teknikertips: Periodisk genomsökning (Defense in Depth)

Som vi nämnde tidigare inaktiveras Microsoft Defender om du installerar ett annat antivirus (t.ex. F-Secure eller CrowdStrike). Men du kan behålla Defender som en "backup" genom att aktivera Periodisk genomsökning.

Så aktiverar du det:

Gå till Windows-säkerhet > Virus- och hotskydd.

Klicka på Alternativ för Microsoft Defender Antivirus.

Slå på Periodisk genomsökning.

Detta gör att Windows kör Defenders motor i bakgrunden då och då för att dubbelkolla systemet, även om ditt huvudskydd missar något. Det är ett klassiskt exempel på att bygga säkerhet i flera lager.

### 8.1.4 Gruppolicy och säkerhetsmallar (Group Policy & Security Templates)

Gruppolicy (Group Policy) är "hjärnan" i en Windows-miljö. Det är det kraftfullaste verktyget en administratör har för att centralt styra hur datorer, konton och säkerhetsinställningar fungerar. Istället för att konfigurera 100 datorer manuellt, skapar du en regel (ett Group Policy Object – GPO) och skickar ut den till alla enheter samtidigt.

#### Verktygen: Var hittar du dem?

Det finns två sätt att hantera gruppolicys beroende på om du jobbar lokalt eller i ett nätverk:

Lokal gruppolicy (gpedit.msc): Används för att styra inställningar på en enskild dator.

Hitta det: Tryck Win + R, skriv gpedit.msc.

Begränsning: Finns endast i Windows Pro, Enterprise och Education.

Group Policy Management Console (GPMC): Används i domänmiljöer (Active Directory) för att styra tusentals datorer.

Hitta det: Finns på domänkontrollanten under Administrative Tools eller genom att skriva gpmc.msc.

#### Hierarki och arv: LSDOU-principen

För att förstå hur policys tillämpas måste en tekniker förstå prioritetsordningen. Om två policys säger emot varandra, vinner den som tillämpas sist. Ordningen kallas LSDOU:

Local (Lokala inställningar på datorn)

Site (Fysisk plats i nätverket)

Domain (Hela domänen)

Organizational Unit (Specifik undergrupp, t.ex. "Elevdatorer") — Högst prioritet.

#### Dator- vs. Användarkonfiguration

Inuti en GPO finns två huvuddelar som en tekniker måste hålla isär:

Computer Configuration: Inställningar som gäller själva hårdvaran. Tillämpas när datorn startar (t.ex. brandväggsregler, BitLocker-krav).

User Configuration: Inställningar som följer med användaren. Tillämpas vid inloggning (t.ex. skrivbordsbakgrund, mappomdirigering).

#### Praktiska exempel på styrning:

Innan vi går in på de tekniska detaljerna, här är vad en tekniker typiskt använder GPO till för att höja säkerheten:

Lösenordspolicy: Tvinga fram längd (minst 12 tecken) och komplexitet.

USB-begränsning: Blockera skrivåtkomst till externa lagringsmedier för att förhindra datastöld.

AppLocker: Bestämma exakt vilka programfiler som får köras (se kapitel 8.1.11).

Windows Update: Styra när datorer ska installera säkerhetspatchar för att undvika driftstopp.

#### Avancerad funktion: Loopback Processing

I vissa fall vill du att användarinställningar ska styras av vilken dator någon sitter vid, snarare än vem personen är.

Exempel: En lärare loggar in på en dator i ett publikt bibliotek. Du vill inte att lärarens alla privata filer och genvägar ska dyka upp där. Genom att aktivera Loopback Processing kan du tvinga datorns specifika begränsningar att trumfa användarens vanliga profil.

#### Säkerhetsmallar (Security Templates)

Att konfigurera hundratals säkerhetsinställningar från grunden är tidskrävande. Därför använder tekniker ofta Säkerhetsmallar (.inf-filer). Detta är färdiga paket med rekommenderade inställningar från Microsoft eller CIS (Center for Internet Security).

#### Hur ser en säkerhetsmall ut?

En säkerhetsmall är en textfil i formatet INI. Den är uppdelad i sektioner som talar om för Windows exakt vilka registervärden eller rättigheter som ska konfigureras.

[System Access]

MinimumPasswordLength = 12

PasswordComplexity = 1

[Registry Values]

MACHINE\Software\Microsoft\Windows\CurrentVersion\Policies\System\DisableCAD=4,0

[System Access]: Här ställs lösenordskraven in.

[Registry Values]: Här ändras registernycklar, t.ex. för att tvinga fram Ctrl+Alt+Del vid inloggning.

#### Arbetsflöde för tekniker

Metod 1: Importera i en GPO (Domän) Högerklicka på Security Settings inuti GPO-editorn och välj Import Policy. Bläddra fram din .inf-fil. Alla inställningar fylls i automatiskt.

Metod 2: Analysera lokalt Använd MMC-snapinet Security Configuration and Analysis. Här kan du välja Analyze Computer Now för att jämföra datorns inställningar mot mallen innan du väljer att applicera dem.

#### Var hittar du de bästa mallarna?

Microsoft Security Compliance Toolkit: Officiella "Security Baselines" för Windows.

CIS (Center for Internet Security): Global standard för härdning av system.

### 8.1.5 Avancerad NTFS-behörighet

NTFS (New Technology File System) är standardfilsystemet i Windows. Dess främsta styrka är den inbyggda säkerheten som gör det möjligt att styra åtkomst på fil- och mappnivå. För en tekniker handlar det inte bara om att välja "Läs" eller "Skriv", utan om att förstå hur rättigheter ärvs, kombineras och förändras när filer flyttas.

#### De vanligaste NTFS-behörigheterna

Rättigheter delas ofta in i "Standard" och "Avancerade". Här är de vanligaste du möter i rollen som administratör:

Behörighet

Vad det innebär i praktiken

Read (Läsa)

Se filnamn, läsa innehåll och se attribut.

Write (Skriva)

Skapa nya filer och mappar samt ändra innehåll i befintliga.

Read & Execute

Samma som läsa, men tillåter även att program (.exe) startas.

Modify (Ändra)

Läsa, skriva och radera filer/mappar.

Full Control

Allt ovan, samt rätten att ändra behörigheter och ta ägarskap.

Viktigt: Skillnaden mellan Modify och Full Control är kritisk. Med Modify kan du göra allt med själva filen, men bara med Full Control kan du bestämma vem andra får ge åtkomst till.

#### Avancerade rättigheter

Tänk på Standardrättigheter som färdiga paketmenyer på en restaurang. När du väljer "Read", får du automatiskt ett knippe av flera små underliggande rättigheter. Avancerade rättigheter är däremot "à la carte" – där kan du plocka exakt de specifika funktioner du vill ha eller neka.

Det finns totalt 14 avancerade rättigheter i NTFS. Här är de viktigaste som en tekniker bör ha koll på:

Avancerad rättighet

Vad den gör (som standardpaketen inte visar)

List Folder / Read Data

Skillnaden mellan att se att en fil finns och att faktiskt få öppna den.

Append Data

Tillåter användaren att lägga till text i slutet av en fil (t.ex. en loggfil) men inte ändra det som redan står där.

Delete Subfolders and Files

Ger rätt att radera filer inuti en mapp även om användaren inte har raderingsrätt på själva filerna individuellt.

Read / Write Attributes

Möjligheten att ändra om en fil ska vara "Skrivskyddad" eller "Dold".

Change Permissions

Denna ingår bara i Full Control. Det är rätten att agera administratör för filens säkerhet.

Take Ownership

Rätten att göra sig själv till ägare av filen. Ägaren kan alltid återta kontrollen, även om någon annan nekat dem åtkomst.

#### När används Avancerade rättigheter?

I 90% av fallen räcker standardrättigheterna, men här är tre scenarier där en tekniker måste gå in i de avancerade inställningarna:

#### 1. Inlämningslådan (The "Drop-box")

En lärare vill att elever ska kunna lämna in sina filer i en mapp, men eleverna får absolut inte se vad klasskamraterna har lämnat in.

Lösning: Ge gruppen "Elever" de avancerade rättigheterna Create Files / Write Data, men neka dem List Folder / Read Data. De kan "posta" brev i lådan, men de kan inte titta ner i den.

#### 2. Loggfiler (Append only)

Ett program ska skriva loggar över vad som händer i systemet. Du vill att programmet ska kunna lägga till nya rader, men en hackare får inte kunna radera eller ändra tidigare loggposter för att dölja sina spår.

Lösning: Ge tjänstekontot rättigheten Append Data, men ta bort Write Data (som tillåter överskrivning).

#### 3. Skydda mappar mot oavsiktlig radering

Du har en mappstruktur som är kritisk. Användare ska få skapa och radera filer inuti mappar, men de får absolut inte råka dra hela huvudmappen till papperskorgen.

Lösning: Ge användarna rättigheter på undernivå, men gå in på huvudmappens avancerade rättigheter och sätt Deny på Delete för just den mappen.

För att se dessa rättigheter räcker det inte med den vanliga säkerhetsfliken:

Högerklicka på mappen > Egenskaper > Säkerhet.

Klicka på knappen Avancerat (Advanced) längst ner.

Markera en användare och klicka på Redigera (Edit).

Klicka på Visa avancerade behörigheter (Show advanced permissions).

#### Arv (Inheritance) – Hur rättigheter flödar

Som standard använder Windows Arv. Det innebär att en fil eller undermapp automatiskt får samma rättigheter som sin "förälder" (mappen den ligger i).

Explicit behörighet: Rättigheter du sätter direkt på en specifik fil eller mapp.

Ärvd behörighet: Rättigheter som "följer med" från mappen ovanför.

När du vill att en undermapp ska ha helt andra regler måste du Inaktivera arv (Disable inheritance). Windows ger dig då två val:

Convert (Omvandla): Behåll de ärvda rättigheterna men gör dem till "explicita" så att du kan redigera dem.

Remove (Ta bort): Rensa mappen på alla ärvda rättigheter och börja från noll.

#### Flytt och kopiering – Vad händer med rättigheterna?

Detta är ett av de viktigaste områdena att bemästra för att undvika att känslig data plötsligt blir åtkomliga för fel personer. Reglerna skiljer sig beroende på om du stannar på samma partition (volym) eller byter till en annan.

#### Logik-matrisen:

Åtgärd

Destination

Resultat

Kopiera

Samma eller annan volym

Ärver från målmappen.

Flytta

Samma volym (t.ex. C: till C:)

Behåller sina ursprungliga rättigheter.

Flytta

Varför behåller flytt på samma volym rättigheterna? Windows ändrar bara "adressen" i filsystemets register, inte själva filobjektet.

Varför ärver flytt mellan olika volymer? Windows måste skriva datan på den nya disken och radera originalet. Det ses som att en ny fil skapas på destinationen.

Varning: Om du flyttar filer till en disk med FAT32 eller exFAT (vanligt på USB-minnen), raderas alla rättigheter helt eftersom dessa filsystem saknar stöd för behörigheter.

#### Effektiva rättigheter, Share och verktyg

När en användare tillhör flera grupper räknar Windows ut de effektiva rättigheterna:

Kumulativ princip: Rättigheter läggs ihop (Läs + Skriv = Läs & Skriv).

Neka (Deny) vinner alltid: En uttrycklig "Neka" trumfar alla "Tillåt".

Share vs. NTFS:

När man ansluter via nätverket kollar Windows på både Share Permissions (delning) och NTFS Permissions. Windows väljer alltid den mest restriktiva inställningen av de två.

### 8.1.6 BitLocker och EFS – Kryptering på disk- och filnivå

Kryptering är sista försvarslinjen för att skydda "data at rest" (lagrad data). Om en obehörig person får fysisk tillgång till en hårddisk – till exempel genom att stjäla en laptop eller ett USB-minne – ser krypteringen till att datan är helt oläsbar utan rätt nyckel. I Windows använder vi främst två tekniker: BitLocker för hela diskar och EFS för enskilda mappar.

#### BitLocker: Skydd för hela systemet

BitLocker krypterar hela partitioner (t.ex. C:). När BitLocker är aktiverat krypteras all data automatiskt i bakgrunden. För att en användare ska kunna starta datorn och läsa datan krävs ett samspel mellan mjukvara och hårdvara.

#### Hur BitLocker samverkar med hårdvaran

I moderna datorer med Windows 11 är BitLocker hårt knutet till säkerhetschippet TPM 2.0 (Trusted Platform Module).

TPM-kontroll: Vid varje start kontrollerar TPM-chippet att systemfilerna och startsekvensen inte har ändrats.

Secure Boot: Om Secure Boot inaktiveras eller om UEFI-inställningarna ändras drastiskt, vägrar TPM-chippet att släppa ifrån sig krypteringsnyckeln.

PIN-kod: För högsta säkerhet kan man kräva en PIN-kod vid uppstart. Detta skyddar mot "Cold boot"-attacker där någon stjäl en dator som nyss varit igång.

#### Vad innebär AES-kryptering (128 vs 256 bitar)?

BitLocker använder AES (Advanced Encryption Standard). Som standard använder Windows en nyckellängd på 128 bitar, men tekniker väljer ofta 256 bitar för maximal säkerhet.

AES-128: Har 2128 möjliga kombinationer. Det skulle ta miljarder år för dagens superdatorer att knäcka detta via "brute force".

AES-256: "Militär säkerhetsnivå". Den innehåller 2256 möjliga kombinationer. Det gör den exponentiellt mycket svårare att knäcka än 128-bitars. Även om det kräver ca 40% mer beräkningskraft av processorn, märks det knappt på moderna datorer.

För att tvinga Windows att använda AES-256 måste man ofta ändra en inställning i Gruppolicy (GPO) innan man startar krypteringen.

#### När krävs återställningsnyckeln?

Det finns ett antal saker som kan "trigga" BitLocker att gå in i Recovery Mode, vilket innebär att den låser sig. Det är en skyddsmekanism för att hindra obehöriga från att komma åt datan på disken. Då krävs en 48 siffrig återställningsnyckel. Exempel på dessa scenarion är:

Hårdvaran ändras: Du flyttar hårddisken till en annan dator för att rädda filer.

Moderkortet byts ut: Eftersom TPM-chippet sitter på moderkortet försvinner nyckeln vid ett hårdvarufel.

Säkerhetsinställningar ändras: Om någon uppdaterar BIOS/UEFI eller ändrar Secure Boot-inställningar utan att först "pausa" (suspend) BitLocker.

#### BitLocker To Go (USB-minnen)

Detta är tekniken för att kryptera flyttbara enheter. Eftersom ett USB-minne inte har tillgång till en dators fasta TPM-chip, används istället ett lösenord för att låsa upp det. Detta är ett absolut krav i de flesta professionella organisationer för att förhindra dataläckage vid borttappad utrustning.

#### Kryptering av enskilda mappar (EFS)

BitLocker skyddar hela "lådan" (disken), men ibland vill man bara låsa vissa "dokument" (mappar) inuti lådan för att skydda dem mot andra användare på samma dator. Här används EFS (Encrypting File System).

Funktion: EFS knyter krypteringen direkt till ditt specifika Windows-användarkonto.

Användning: Högerklicka på en mapp > Egenskaper > Avancerat > Markera "Kryptera innehåll för att säkra data".

### 8.1.7 Windows-brandväggen med avancerad säkerhet

Windows-brandväggen fungerar som ett filter mellan operativsystemet och nätverket. Dess uppgift är att inspektera varje datapaket som försöker passera och avgöra om det ska tillåtas eller blockeras baserat på ett logiskt regelverk. För professionell administration används verktyget Windows-brandväggen med avancerad säkerhet (Windows Defender Firewall with Advanced Security).

#### Hur verktyget nås

Det finns flera vägar in i det avancerade gränssnittet, beroende på arbetsflöde:

Kör-kommandot: Tryck Win + R och skriv wf.msc. Detta är den snabbaste vägen för en tekniker.

Sök: Skriv "Windows-brandväggen med avancerad säkerhet" i Start-menyn.

Kontrollpanelen: Navigera till System och säkerhet > Windows Defender-brandväggen > Avancerade inställningar.

När en regel skapas behöver administratören avgöra hur specifik styrningen ska vara.

Programbaserade regler: Här tillåts eller blockeras en specifik körbar fil (t.ex. sqlservr.exe).

Fördel: Brandväggen håller reda på vilka portar programmet behöver dynamiskt. Det är säkrare för applikationer som byter portar ofta.

Portbaserade regler: Här styrs trafiken baserat på TCP- eller UDP-portar.

Fördel: Ger exakt kontroll över nätverkstrafiken oavsett vilket program som lyssnar. Detta används främst för webbservrar (port 80/443), filservrar (port 445) eller fjärrstyrning (port 3389).

En av de mest kraftfulla funktionerna i den avancerade brandväggen är Scope. Det gör det möjligt att definiera exakt vilka IP-adresser en regel ska gälla för.

Lokal IP-adress: Används om datorn har flera nätverkskort och regeln bara ska gälla ett av dem.

Fjärr-IP-adress: Här kan administratören begränsa åtkomst så att t.ex. fjärrskrivbord (RDP) endast tillåts från kontorets specifika IP-adress-intervall, istället för från hela internet. Detta minskar attackytan drastiskt.

#### Stateful Packet Inspection (SPI)

Windows-brandväggen är "stateful". Det innebär att den förstår sammanhanget i en kommunikation. Om datorn påbörjar en utgående session (t.ex. öppnar en hemsida), kommer brandväggen ihåg detta och tillåter automatiskt den inkommande trafiken som tillhör just den sessionen. Detta eliminerar behovet av att manuellt öppna portar för svarstrafik.

#### Nätverksprofiler

Brandväggen applicerar olika regeluppsättningar beroende på nätverkets status:

Domän (Domain): För datorer anslutna till Active Directory.

Privat (Private): För betrodda nätverk (hem/kontor).

Offentlig (Public): Den mest restriktiva profilen där datorn görs osynlig för andra enheter.

#### Loggning och felsökning

Vid problem med nätverkskommunikation kan brandväggens logg aktiveras för att se exakt vilka paket som kastas bort. Loggfilen hittas normalt under: %systemroot%\system32\LogFiles\Firewall\pfirewall.log

### 8.1.8 Autentisering: Kerberos, NTLM, Windows Hello och SSO

Autentisering är den tekniska processen där operativsystemet fastställer att en användare eller enhet faktiskt är den de utger sig för att vara. I en modern Windows-miljö samverkar flera protokoll för att möjliggöra säker inloggning och sömlös åtkomst till resurser via Single Sign-On (SSO).

#### Kerberos – Standardprotokollet i domänmiljö

Kerberos är det primära autentiseringsprotokollet i Active Directory. Det eliminerar behovet av att skicka lösenord över nätverket genom att använda ett system baserat på kryptografiska biljetter.

Mekanismen bakom Kerberos: Processen hanteras centralt av tjänsten Key Distribution Center (KDC), som körs på nätverkets domänkontrollanter.

Ticket-Granting Ticket (TGT): Vid inloggning skickas en krypterad begäran till KDC:n. Om autentiseringen lyckas utfärdas en TGT. Denna fungerar som användarens digitala legitimation under resten av sessionen.

Service Tickets: Varje gång användaren vill nå en resurs, till exempel en filserver eller en databas, presenteras användarens TGT för KDC:n. KDC:n utfärdar då en specifik tjänstebiljett för just den resursen. Detta möjliggör Single Sign-On (SSO), då användaren inte behöver ange sina uppgifter på nytt för varje resurs.

Säkerhetsfördelar: Kerberos använder tidsstämplar för att förhindra "Replay-attacker", där en angripare försöker fånga upp och återanvända autentiseringsdata. Detta kräver att klockorna i nätverket är synkroniserade (oftast inom en marginal på 5 minuter).

#### NTLM (New Technology LAN Manager)

NTLM är ett äldre protokoll som baseras på en "utmaning och svar"-logik (Challenge/Response). Det fungerar idag främst som ett reservalternativ (fallback) eller för autentisering i nätverk som saknar en domänstruktur.

Säkerhetsutmaningar med NTLM:

Protokollsvagheter: NTLM är sårbart för Relay-attacker och Pass-the-Hash. Vid en Pass-the-Hash-attack behöver en angripare inte känna till det faktiska lösenordet, utan kan använda den sparade kryptografiska hashen för att bevisa sin identitet mot andra servrar.

Brist på ömsesidig autentisering: Till skillnad från Kerberos bevisar inte servern sin identitet för klienten i samma utsträckning, vilket ökar risken för "Man-in-the-middle"-attacker.

På grund av dessa brister strävar många organisationer efter att minimera eller helt blockera NTLM-trafik genom säkerhetspolicyer.

#### Windows Hello for Business

Windows Hello representerar ett skifte från lösenordsbaserad inloggning till asymmetrisk kryptografi knuten till hårdvaran.Istället ersätter Windows Hello lösenordet med en privat och en offentlig nyckel.

Lokal verifiering: Användaren verifierar sin identitet lokalt mot enheten via biometri eller en PIN-kod. Denna handling låser upp den privata nyckeln.

TPM-integration: Den privata nyckeln lagras säkert i datorns TPM-chip och lämnar aldrig enheten. Detta innebär att en angripare som kommer över användarens PIN-kod inte kan använda den på en annan dator.

FIDO2 och MFA: Tekniken bygger på öppna standarder som FIDO2. Eftersom inloggningen kräver både en specifik fysisk enhet (datorn) och en biometrisk verifiering, fungerar Windows Hello som en inbyggd multifaktorautentisering (MFA).

#### Single Sign-On (SSO) i hybridmiljöer

För att användare ska kunna nå molntjänster (t.ex. Microsoft 365) utan att logga in på nytt, bryggas den lokala autentiseringen med molnbaserad identitetshantering.

Primary Refresh Token (PRT): På enheter som är anslutna till Azure AD (Entra ID) utfärdas en PRT vid inloggning. Denna fungerar som molnets motsvarighet till en Kerberos TGT och möjliggör SSO för webbapplikationer och molnresurser.

#### Verktyg för administration och analys

Administratörer använder följande verktyg för att styra och övervaka autentiseringsflöden:

Lokal säkerhetspolicy (secpol.msc): Används för att konfigurera begränsningar för NTLM och ställa in krav för Windows Hello.

Loggboken (Event Viewer): Under Windows-loggar > Säkerhet loggas inloggningshändelser. Händelse-ID 4624 (lyckad inloggning) visar vilket protokoll (Kerberos, NTLM eller annat) som användes i det specifika fallet.

klist: Ett kommandoradsverktyg som används för att lista och hantera de Kerberos-biljetter som för tillfället finns i datorns minne. Genom att skriva klist i terminalen kan en tekniker verifiera att rätt biljetter har utfärdats.

### 8.1.9 Revision och Loggning (Auditing)

Revision (Auditing) är en reaktiv säkerhetsåtgärd som syftar till att dokumentera händelser i operativsystemet. Medan brandväggar och antivirus fungerar som hinder, fungerar loggningen som systemets övervakningskamera. Utan en korrekt konfigurerad revision är det svårt att i efterhand genomföra en forensisk analys av ett intrång eller utreda interna säkerhetsincidenter.

#### Granskningsprinciper (Audit Policies)

Innan Windows börjar logga specifika händelser krävs att en Granskningsprincip (Audit Policy) aktiveras. Detta görs oftast via Lokal säkerhetspolicy (Local Security Policy - secpol.msc) eller centralt via Grupprincipshantering (Group Policy Management - GPMC). Man skiljer mellan två typer av registreringar:

Lyckade händelser (Success): Loggar när en åtgärd genomförts korrekt (t.ex. en användare loggar in).

Misslyckade händelser (Failure): Loggar när en åtgärd nekas (t.ex. någon försöker öppna en mapp de inte har behörighet till).

Viktiga kategorier att granska:

Inloggningshändelser (Logon Events): Dokumenterar varje försök att ansluta till datorn. Misslyckade försök kan indikera ett pågående angrepp via råstyrka (Brute force).

Objektåtkomst (Object Access): Loggar när någon läser, ändrar eller raderar specifika filer och mappar.

Behörighetsanvändning (Privilege Use): Registrerar när en användare utnyttjar administrativa rättigheter (t.ex. ändrar systemtiden eller tar ägarskap över filer).

#### Loggboken (Event Viewer)

Loggboken (Event Viewer) är det primära gränssnittet för att läsa systemets händelser. Loggarna delas upp i flera kategorier:

Säkerhetsloggen (Security Log): Innehåller all data som definierats i granskningsprinciperna. Detta är den mest kritiska loggen för säkerhetsanalys.

Systemloggen (System Log): Dokumenterar händelser relaterade till Windows-komponenter och drivrutiner (t.ex. om en tjänst misslyckas att starta).

Programloggen (Application Log): Innehåller händelser från installerad mjukvara.

Kritiska Händelse-ID (Event IDs): Genom att filtrera på specifika ID-nummer kan incidenter identifieras snabbt:

4624: Lyckad inloggning (Successful Logon).

4625: Misslyckad inloggning (Failed Logon).

4663: Ett objekt (t.ex. en fil) har öppnats, skrivits till eller raderats.

4720: Ett nytt användarkonto har skapats.

#### Loggcentralisering (Event Forwarding)

I större nätverk innebär det en säkerhetsrisk att endast lagra loggar lokalt på varje enskild dator. Om en angripare tar kontroll över en maskin är ett vanligt första steg att radera loggarna för att dölja sina spår. För att motverka detta tillämpas Vidarebefordring av händelser (Event Forwarding).

Mekanism: Klientdatorer konfigureras att skicka sina viktigaste logghändelser till en central server (en Collector).

Säkerhetsfördel: Även om en lokal logg raderas, finns bevisningen kvar på den centrala servern. Detta underlättar också korrelering, där man kan se om samma IP-adress försöker logga in på flera datorer samtidigt.

#### Praktiskt scenario: Spåra filradering

För att identifiera vem som raderat en känslig fil på en filserver krävs två steg:

Global aktivering: Granskningsprincipen för Objektåtkomst (Object Access) måste vara aktiverad i säkerhetspolicyn.

Specifik aktivering: På själva mappen måste inställningar för Granskning (Auditing) konfigureras under Egenskaper > Säkerhet > Avancerat. Här definieras att systemet ska logga raderingar för en specifik grupp, exempelvis "Alla" (Everyone).

### 8.1.10 Skydd mot skadlig kod (Malware) och nätfiske (Phishing)

Skyddet mot externa hot i en modern Windows-miljö bygger på en kombination av proaktiv analys, ryktesbaserade system och isolering. Eftersom en överväldigande majoritet av intrång påbörjas via social manipulation, är försvaret integrerat i allt från e-postfilter till operativsystemets kärna.

#### Kategorisering av skadlig kod (Malware)

Skadlig kod (Malware) är ett samlingsnamn för programvara med illasinnade avsikter. För en administratör är det viktigt att förstå spridningsmönstren för att kunna applicera rätt motåtgärder.

Typ

Teknisk beskrivning

Virus (Virus)

Kod som infekterar legitima filer och kräver användarinteraktion (t.ex. att en fil öppnas) för att sprida sig.

Mask (Worm)

Självreplikerande kod som utnyttjar nätverkssårbarheter (t.ex. SMB-brister) för att sprida sig autonomt mellan system.

Trojan (Trojan)

Maskerad som legitim mjukvara. Den skapar ofta en bakdörr (Backdoor) som ger angriparen fjärråtkomst.

Utpressningstrogen (Ransomware)

Krypterar användarens filer med asymmetrisk kryptering (t.ex. AES-256) och kräver lösensumma för dekrypteringsnyckeln.

Spionprogram (Spyware)

Exfiltrerar data i tystnad, såsom tangenttryckningar (Keylogging) eller webbhistorik, till en extern server (C2-server).

#### Nätfiske (Phishing) och social manipulation

Nätfiske (Phishing) syftar till att stjäla identiteter eller installera skadlig kod genom att utnyttja mänskliga faktorer. Angreppssätten har blivit alltmer sofistikerade:

Riktat nätfiske (Spear Phishing): Ett skräddarsytt angrepp mot en specifik individ eller organisation, ofta baserat på insamlad information från sociala medier.

SMS-fiske (Smishing): Bedrägeriförsök via SMS som ofta innehåller brådskande uppmaningar om paketleveranser eller bankärenden.

Röstfiske (Vishing): Bedrägeri via telefonsamtal där angriparen ofta använder social ingenjörskonst (Social Engineering) för att få offret att utföra åtgärder via BankID eller fjärrstyrningsprogram.

#### Tekniska skyddsmekanismer i Windows

För att motverka dessa hot använder Windows flera lager av försvar som opererar oberoende av varandra.

#### Microsoft Defender SmartScreen (Microsoft Defender SmartScreen)

SmartScreen är ett ryktesbaserat skydd som verifierar webbplatser och filer mot Microsofts globala molndatabas.

Analys av webbplatser: Om en användare besöker en känd nätfiskesida blockeras åtkomsten omedelbart.

Filsignering: Filer som saknar en digital signatur från en betrodd utgivare eller som är ovanliga (lågt rykte) flaggas med en varning.

#### Realtidsskydd och heuristik (Real-time Protection & Heuristics)

Det inbyggda antivirusprogrammet förlitar sig inte bara på kända virusdefinitioner (Signaturer).

Beteendeanalys (Behavioral Monitoring): Övervakar processer i realtid. Om ett program plötsligt börjar kryptera filer i hög takt (typiskt för Ransomware), stoppas processen omedelbart.

Heuristik (Heuristics): Möjliggör identifiering av ny skadlig kod genom att leta efter kodsekvenser som påminner om kända hot, även om den exakta filen aldrig sett förut.

#### Attackytans begränsning (Attack Surface Reduction - ASR)

ASR är en uppsättning regler som blockerar beteenden som ofta används av skadlig kod, exempelvis:

Blockering av körbara filer från e-postklienter.

Blockering av Office-makron som försöker skapa nya processer.

Skydd mot injicering av kod i andra processers minnesutrymme.

#### Isolerad webbläsning (Application Guard)

För högsäkerhetsmiljöer kan Microsoft Defender Application Guard användas. Det öppnar webbläsaren i en isolerad hårdvarubaserad behållare (Sandbox). Om användaren råkar besöka en infekterad sida stannar den skadliga koden inuti behållaren och raderas när webbläsaren stängs, utan att påverka resten av operativsystemet.

#### Administration via Grupprinciper (GPO)

I en domänmiljö konfigureras dessa skydd centralt för att säkerställa att slutanvändare inte kan inaktivera dem. Administratören kan tvinga fram inställningar som:

Obligatorisk molnbaserad genomsökning.

Automatiska definitionshämtningar flera gånger per dygn.

Aktivering av Kontrollerad mappåtkomst (Controlled Folder Access) för att skydda dokumentmappar mot Ransomware.

### 8.1.11 PowerShell – Arkitektur, användning och säkerhet

PowerShell är en objektorienterad automatiseringsplattform och ett skriptspråk byggt på .NET-ramverket. Till skillnad från äldre gränssnitt som Kommandotolken (Command Prompt - cmd.exe), som enbart hanterar textsträngar, arbetar PowerShell med objekt. Detta innebär att varje kommando returnerar strukturerad data som kan manipuleras och skickas vidare med extrem precision.

#### Vad är PowerShell?

PowerShell består av en kommandoradsmiljö (Command-Line Interface - CLI) och ett skriptspråk. Det är utformat för att hantera allt från enstaka datorer till hela molninfrastrukturer.

Kommandon (Cmdlets): PowerShell använder en konsekvent syntax baserad på Verb-Noun (t.ex. Get-Service eller Stop-Process).

Pipelinen (The Pipeline): Med tecknet | skickas hela objekt från ett kommando till nästa. Om man kör Get-Process | Sort-Object CPU, skickas faktiska processobjekt vidare för sortering, inte bara textrader.

Fjärrstyrning (PowerShell Remoting): Genom protokollet WinRM (Windows Remote Management) kan en administratör köra kommandon på hundratals servrar samtidigt från sin egen arbetsstation.

#### Hur PowerShell används i administrationen

Inom IT-drift används PowerShell för att eliminera repetitiva uppgifter och säkerställa en enhetlig konfiguration.

Automatisering (Automation): Skript som körs schemalagt för att exempelvis rensa loggar, skapa användarkonton eller kontrollera diskutrymme.

Konfigurationshantering (Configuration Management): Verktyg som Desired State Configuration (DSC) gör det möjligt att definiera hur en server ska se ut i en textfil, varpå PowerShell ser till att servern bibehåller det tillståndet.

#### PowerShell, säkerhetsrisk?

Begreppet "Living off the Land" används ofta för att beskriva hur angripare utnyttjar legitima verktyg som redan finns i operativsystemet. Eftersom PowerShell har djup tillgång till operativsystemets kärna och ofta körs med höga rättigheter, är det ett primärt mål för attacker.

Minnesbaserade attacker (Fileless Malware): Skadlig kod kan laddas ner direkt från internet och köras i RAM-minnet via PowerShell utan att en enda fil någonsin skrivs till hårddisken. Detta gör att traditionella antivirusprogram ofta missar aktiviteten.

Obfuskering (Obfuscation): PowerShell-kod kan skrivas så att den är nästintill oläslig för människor men fullt körbar för datorn, vilket döljer angriparens avsikter.

#### Säkerhetsmekanismer och skyddsåtgärder

Inledningsvis är det viktigt att förstå att Windows erbjuder ett flertal inbyggda säkerhetsfunktioner för att balansera den enorma kraften i PowerShell mot behovet av systemhärdning. Dessa mekanismer fungerar inte som enskilda lösningar, utan snarare som kompletterande lager som tillsammans minskar risken för att verktyget missbrukas av skadlig kod eller obehöriga aktörer. Genom att implementera dessa skydd kan en organisation dra nytta av automatiseringens fördelar utan att exponera systemet för onödiga risker.

En av de mest grundläggande kontrollerna är Körningsprincipen (Execution Policy). Det är viktigt att notera att detta i första hand fungerar som ett skydd mot att skript körs av misstag, snarare än en ogenomtränglig säkerhetsgräns för en medveten angripare. Genom läget Restriktiv (Restricted) kan systemet konfigureras att blockera samtliga skriptfiler, medan läget Fjärrsignerad (RemoteSigned) tillåter lokalt skrivna skript men kräver digital signering för allt som laddats ner från externa källor. I miljöer med extremt höga säkerhetskrav används ofta Allt signerat (AllSigned), vilket tvingar fram en strikt kontroll där varje enskilt skript måste vara signerat av en betrodd utgivare för att få exekveras.

För att uppnå en god spårbarhet och möjliggöra efterhandsanalyser vid incidenter tillämpas ofta utökad Loggning och spårbarhet (Logging and Traceability). Inom ramen för detta är Loggning av skriptblock (Script Block Logging) en kritisk komponent, då den registrerar det faktiska innehållet i de kommandon som körs, även om koden har varit dold eller krypterad (Obfuscated) för att undvika upptäckt. Detta kan kombineras med Transkribering (Transcription), vilket genererar en detaljerad textbaserad historik över hela sessionen, inklusive alla inmatade kommandon och deras resultat. Detta ger en fullständig bild av administratörens – eller angriparens – aktivitet i terminalen.

Ytterligare en nivå av skydd kan uppnås genom att aktivera Begränsat språkläge (Constrained Language Mode). I detta läge begränsas de tekniska möjligheterna i språket genom att blockera åtkomst till avancerade .NET-klasser och Windows-API:er, vilka ofta är en förutsättning för att skadlig kod ska kunna manipulera systemets kärnfunktioner. Denna funktion används med fördel tillsammans med andra lösningar för applikationskontroll, såsom AppLocker, för att skapa en strikt och kontrollerad miljö där endast de absolut nödvändiga funktionerna är tillgängliga för användaren eller skriptet.

Slutligen fungerar Gränssnittet för genomsökning av skadlig kod (Anti-Malware Scan Interface - AMSI) som en vital brygga mellan PowerShell och systemets antiviruslösning. AMSI fungerar genom att inspektera koden precis i det ögonblick den ska exekveras, efter att eventuell döljande kod har packats upp i arbetsminnet (RAM). Detta gör det möjligt för Windows att identifiera och blockera även Filfri skadlig kod (Fileless Malware) som aldrig lagras på hårddisken. På så sätt kan säkerhetssystemet fatta proaktiva beslut baserat på programmets faktiska logik snarare än dess yttre form.

### 8.1.12 Användarkonton och grupper

Hanteringen av användarkonton (User Accounts) och grupper (Groups) är fundamentet för all säkerhet i Windows. Genom att korrekt definiera vem en användare är och vilka rättigheter denna har, kan man begränsa skadan vid en eventuell kompromettering. En grundläggande princip inom säkerhet är Principen om minsta möjliga behörighet (Principle of Least Privilege - PoLP), vilket innebär att en användare aldrig ska ha mer rättigheter än vad som krävs för att utföra sina arbetsuppgifter.

#### Olika typer av användarkonton

Windows använder olika kontotyper för att separera administrativa uppgifter från vardagligt arbete. Valet av kontotyp påverkar direkt systemets attackyta.

Kontotyp

Beskrivning och rättigheter

Vanligt användningsområde

Administratör (Administrator)

Innehar full kontroll över operativsystemet. Kan installera mjukvara, ändra systeminställningar och hantera andra användare.

Systemadministratörer och IT-tekniker vid specifika underhållsinsatser.

Standardkonto (Standard User)

Kan använda installerade program och spara filer i egna mappar, men saknar rättighet att utföra systemförändringar eller installera drivrutiner.

Den rekommenderade kontotypen för det dagliga arbetet (t.ex. för elever eller kontorsanställda).

Gästkonto (Guest Account)

Ett mycket begränsat konto för tillfälliga användare. Har ingen tillgång till personliga filer eller permanenta inställningar.

Oftast inaktiverat i moderna miljöer av säkerhetsskäl.

Microsoft-konto (Microsoft Account)

Ett onlinebaserat konto kopplat till Microsofts molntjänster. Möjliggör synkronisering av inställningar och inloggning i tjänster som OneDrive.

Vanligt i hemmiljöer eller mindre organisationer utan en lokal domänstruktur.

#### Gruppbaserad behörighetshantering

En effektiv administration bygger på att rättigheter tilldelas grupper snarare än enskilda individer. Detta förenklar hanteringen avsevärt när personal byter roller eller slutar.

Inbyggda grupper (Built-in Groups): Windows innehåller färdiga grupper som Administratörer (Administrators), Användare (Users) och Fjärrskrivbordsanvändare (Remote Desktop Users). Genom att placera ett konto i en specifik grupp ärvs alla tillhörande rättigheter automatiskt.

Säkerhetsgrupper (Security Groups): I en domänmiljö skapas ofta egna grupper (t.ex. "Ekonomi" eller "IT-support") för att styra åtkomst till specifika mappar på en filserver.

#### Säkerhetsrekommendationer för kontohantering

För att upprätthålla en hög säkerhetsnivå är det god praxis att följa ett antal etablerade riktlinjer:

Separata konton för administratörer: Även IT-personal bör använda ett standardkonto för vardagliga uppgifter som e-post och webbsurfning. Administrativa rättigheter ska endast användas när det är absolut nödvändigt.

Inaktivera onödiga konton: Gästkonton och gamla användarkonton som inte längre används utgör en säkerhetsrisk och bör därför hållas inaktiverade.

Undvik delade konton: Varje användare bör ha ett unikt konto. Delade konton (t.ex. ett Microsoft-konto för en hel avdelning) leder till bristande spårbarhet och ökar risken för integritetsläckage.

#### Verktyg för administration

Beroende på miljöns komplexitet används olika verktyg för att hantera konton:

Datorhantering (Computer Management - lusrmgr.msc): Det klassiska gränssnittet för att hantera lokala användare och grupper på en enskild dator.

Inställningar (Settings): Det moderna gränssnittet för enklare hantering av lokala konton och Microsoft-konton.

PowerShell: Genom modulen Microsoft.PowerShell.LocalAccounts kan tekniker effektivt skapa och administrera konton via kommandon, exempelvis Get-LocalUser eller New-LocalGroup.
