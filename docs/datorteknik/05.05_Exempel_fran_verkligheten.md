6 Windows

Windows är idag världens mest spridda operativsystem för persondatorer. Men resan dit har varit lång och innehållsrik. För att förstå dagens Windows 11 (och även Windows Server) är det bra att ha en grundläggande bild av hur systemet har utvecklats.

6.1 Microsofts OS-historia

### 6.1.1 DOS-eran

I början av 1980-talet styrdes datorer främst genom att man skrev kommandon i textform i operativsystemet MS-DOS. När de första versionerna av Windows (1.0, 2.0 och det mer framgångsrika 3.x) lanserades, var de inte egna operativsystem i teknisk mening. De fungerade snarare som ett grafiskt "skal" som kördes ovanpå DOS.

Det unika med dessa tidiga versioner var att användaren för första gången kunde hantera filer och program genom att klicka på ikoner och flytta runt fönster istället för att memorera textkommandon. Skillnaden mot det rena DOS-läget var enorm – plötsligt kunde man ha flera program öppna samtidigt (så kallad kooperativ multitasking).

Du kan fortfarfande idag vissa kvarlevor i moderna Windows. Om du öppnar Kommandotolken (cmd.exe) i Windows 11 idag, ser du i princip ett eko av det gamla MS-DOS. Många av kommandona, som dir för att lista filer eller cd för att byta mapp, fungerar exakt likadant nu som 1985.

6.1.2 Windows 95, 98 och ME

Mitten av 90-talet markerade en revolution för persondatorn. Med lanseringen av Windows 95 integrerades DOS och det grafiska gränssnittet i ett mer sammanhållet system än någonsin tidigare. Det var här Microsoft introducerade Start-menyn och Aktivitetsfältet – två funktioner som blivit så fundamentala att vi idag har svårt att föreställa oss en dator utan dem. Gränssnittet blev mer intuitivt, och för första gången kändes Windows som ett riktigt operativsystem snarare än bara ett skal. Windows 98 byggde vidare på denna succé genom att förfina tekniken, introducera bättre stöd för nätverk och den då helt nya USB-standarden. Det var under denna tid som Windows blev "folkligt" och Plug and Play blev ett begrepp – idén att man skulle kunna koppla in hårdvara utan att behöva konfigurera krångliga inställningar manuellt.

Som en sista och något kaotisk avslutning på den här arkitekturen släpptes Windows Millennium Edition (ME) år 2000. Windows ME är historiskt ökänt och anses ofta vara ett av de sämsta operativsystemen genom tiderna; det var ökänt buggigt, instabilt och drabbat av enorma säkerhetshål som gjorde att datorer ofta låste sig utan förvarning. Trots detta var det ett avgörande tekniskt avstamp: det var det första konsument-Windows som försökte kapa navelsträngen till MS-DOS helt. Microsoft blockerade möjligheten att starta datorn i ett "äkta" DOS-läge, vilket signalerade slutet för det textbaserade arvet och början på en framtid där Windows tog full kontroll över hårdvaran från första sekunden.

Windows ME introducerade också flera funktioner som var tänkta att göra livet enklare för hemmanvändaren, bland annat Systemåterställning (System Restore), som lät användaren backa datorns inställningar om något gick fel, och Windows Movie Maker. Men på grund av den instabila koden blev upplevelsen för många användare en mardröm av blåskärmar och krascher. Det blev tydligt att den gamla 9x-arkitekturen hade nått vägs ände, vilket banade väg för den betydligt stabilare NT-kärnan som snart skulle ta över i och med Windows XP.

Start-menyn och aktivitetsfältet är de mest uppenbara arven från 95/98-eran. Men även funktionen Systemåterställning, som föddes i det utskällda Windows ME, lever kvar i högsta grad i Windows 11 som ett sista skyddsnät. Dessutom ser vi än idag filstrukturen med mappen "Program Files" och användningen av genvägar (.lnk-filer) som etablerades under dessa år. Det hade också ett mycket bättre stöd för 32-bitars program (inte lika vanligt idag), men mer integrerade nätverksmöjligheter.

6.1.3 NT-linjen: Grunden för stabilitet

Parallellt med konsumentversionerna utvecklade Microsoft Windows NT (New Technology). Medan Windows 95 ofta kraschade ("Blue Screen of Death" var vardagsmat), byggdes NT för företag som krävde total stabilitet och säkerhet. NT kördes inte ovanpå DOS utan hade en helt egen, modern kärna (kernel). Även om mycket såg lika ut gentemot Windows 95/98/ME, så skiljde sig operativsystemen fundamentalt.

Windows NT introducerade filsystemet NTFS, vilket tillät avancerade rättigheter på filer och mappar – något som är standard än idag. NT 4.0 och senare Windows 2000 lade grunden för hur moderna nätverk och användarkonton hanteras.

Nästan hela grundarkitekturen i moderna Windows är byggd på NT. Varje gång du ser mappen "System32" eller använder "Aktivitetshanteraren" (Ctrl+Alt+Del), rör du vid tekniker som föddes i NT-linjen.

6.1.4 Windows XP

År 2001 hände något avgörande: Microsoft skrotade den gamla osäkra DOS-baserade tekniken för hemmanvändare och flyttade över alla till NT-kärnan. Resultatet blev Windows XP.

XP skilde sig från Windows 98/ME genom att vara oerhört mycket stabilare och ha bättre stöd för multimedia. Det var det första systemet som kändes modernt på riktigt, med stöd för flera användare som kunde vara inloggade samtidigt och ett färgstarkt gränssnitt. XP blev så populärt att det tog över ett decennium för många företag att våga lämna det.

Funktionen för Fjärrskrivbord (Remote Desktop) och tekniken för att jämna ut text på skärmen (ClearType) blev standardiserade i XP och används än idag.

6.1.5 Vista och Windows 7

Efter den långa XP-eran lanserade Microsoft Windows Vista, ett operativsystem som på papperet var en teknisk triumf men som i verkligheten möttes av hård kritik. Vista introducerade ett helt nytt, glasliknande utseende kallat Aero och en rad djupgående säkerhetsfunktioner. Den mest märkbara var UAC (User Account Control), som ständigt avbröt användaren med frågor om lov för att utföra administrativa uppgifter. Detta gjordes för att hindra skadlig kod från att installera sig i smyg, men det ställde till med stora problem. Plötsligt krävde systemet att alla drivrutiner och program var digitalt signerade för att få köras. Det gjorde att äldre hårdvara slutade fungera och att tunga program som tidigare versioner av Adobe Photoshop blev helt okörbara, vilket skapade en enorm frustration hos användarna.

Tre år senare, 2009, lanserades Windows 7 som i princip var en finslipad och optimerad version av Vista. Microsoft hade lyssnat på kritiken: man behöll de kraftfulla säkerhetsfunktionerna men gjorde dem mindre påträngande och optimerade koden så att systemet blev betydligt snabbare och mer stabilt. Windows 7 lyckades med konststycket att vara både modernt och användarvänligt, vilket gjorde att det snabbt blev extremt populärt. Många tekniker och entusiaster anser än idag att Windows 7 är den bästa versionen av Windows som någonsin gjorts, tack vare dess perfekta balans mellan prestanda, drivrutinssupport och ett logiskt gränssnitt.

Det var i Windows 7 som den lilla sökrutan i Start-menyn blev riktigt effektiv och snabb, en funktion som vi idag tar för helt given när vi blixtsnabbt vill hitta ett program eller en fil. Från Vista lever den karaktäristiska UAC-rutan kvar – den som tonar ner hela skärmen och kräver din uppmärksamhet när du ska installera ett nytt program. Även om den inte frågar lika ofta idag, är tekniken bakom (Secure Desktop) en direkt arvtagare från Vistas fokus på att skydda systemkärnan från obehöriga ändringar.

6.1.6 Windows 8 och 10

Efter succén med Windows 7 tog Microsoft ett av sina mest riskfyllda beslut någonsin. Man förutsåg en framtid där pekskärmar skulle dominera allt, från surfplattor och telefoner till arbetsdatorer. Resultatet blev Windows 8, som lanserades med det radikala Metro-gränssnittet (senare kallat Modern UI). Den mest chockerande förändringen var att den klassiska Start-menyn var helt borta. Istället möttes användaren av en startskärm i fullskärmsläge fylld med färgglada "brickor" (Live Tiles) som uppdaterades i realtid.

Tanken var att skapa en sömlös upplevelse; oavsett om du använde en stationär dator, en Microsoft Surface-platta eller en Windows Phone, skulle gränssnittet vara detsamma. Men för de miljoner användare som satt med mus och tangentbord blev Windows 8 en frustrerande upplevelse. Funktioner var dolda i "Charms-menyer" som krävde svepande rörelser, och program öppnades ofta i fullskärmsläge utan enkla sätt att stänga dem. Microsoft hade underskattat hur djupt rotat arbetssättet med skrivbord och fönster var.

Det dröjde inte länge förrän Microsoft insåg att världen inte var redo att kasta bort musen. Redan året efter släpptes Windows 8.1, en uppdatering som ofta beskrivs som en "panikåtgärd". Här tvingades man backa på flera punkter: den lilla Start-knappen kom tillbaka i aktivitetsfältet (även om den fortfarande bara öppnade startskärmen), och man lade till möjligheten att starta datorn direkt till det klassiska skrivbordet istället för till brickorna. Det var ett erkännande att strategin med "touch först" hade gått för långt, för snabbt.

År 2015 lanserades Windows 10, vilket blev den stora räddningen för Microsofts rykte. Man behöll den moderna tekniken under huven men återinförde Start-menyn i en hybridform där den klassiska listan samsades med de moderna brickorna. Windows 10 introducerades som en "tjänst" (Windows as a Service), vilket innebar att Microsoft slutade sälja nya versioner i paket vart tredje år. Istället fick användarna gratis funktionsuppdateringar två gånger om året. Säkerheten fick också ett enormt lyft då Windows Defender blev en fullvärdig och inbyggd antiviruslösning, och Microsoft Store introducerades för att skapa en säkrare plats att ladda ner program på.

Den tydligaste och mest irriterande kvarlevan från den här splittrade eran är uppdelningen av systeminställningar. I Windows 8 introducerades appen Inställningar för att passa pekskärmar, men den saknade många av de djupare verktygen. Detta skapade en rörig verklighet där användaren ständigt fick hoppa mellan den moderna Inställningar-appen och den gamla Kontrollpanelen från NT-tiden. Trots att vi nu är inne i Windows 11-eran lever denna uppdelning kvar, och tekniker måste fortfarande ofta leta i den gamla Kontrollpanelen för att hitta avancerade nätverks- eller drivrutinsinställningar som ännu inte flyttats över.

6.1.7 Windows 11

Windows 11, som lanserades 2021, markerar nästa stora steg i Windows-familjens evolution. Det första som möter användaren är det radikalt förändrade utseendet: Start-menyn har flyttats till mitten av aktivitetsfältet, fönstren har fått rundade hörn och hela systemet känns mer avskalat och lugnt. Men under den polerade ytan döljer sig den mest betydande förändringen på över ett decennium – en stenhård gränsdragning när det gäller säkerhet och hårdvara.

Den största nyheten, och den mest diskuterade, är kravet på TPM 2.0 (Trusted Platform Module) och Secure Boot. Som vi lärde oss i kapitel 2 (Uppstart), handlar Secure Boot om att kontrollera att ingen skadlig kod har manipulerat datorn innan operativsystemet startar. TPM 2.0 tar detta ett steg längre. Man kan likna TPM vid ett litet, fysiskt kassaskåp inbyggt på moderkortet (eller emulerat i processorn). I detta skåp lagras kryptografiska nycklar, lösenord och certifikat på ett sätt som gör dem nästan omöjliga att stjäla digitalt. Genom att kräva TPM 2.0 har Microsoft flyttat säkerhetens grundlinje från mjukvara till hårdvara. Detta har gjort att många fullt fungerande datorer från Windows 10-eran inte kan uppgraderas officiellt, vilket skapat en hel del debatt i teknikvärlden.

När det gäller det dagliga arbetet har Windows 11 introducerat smarta funktioner som Snap Layouts, vilket gör det enkelt att organisera många öppna fönster i färdiga mallar på skärmen – en räddare i nöden för de som jobbar med stora skärmar. Microsoft har också lagt ner mycket arbete på att städa upp i det röriga arvet från Windows 8. Appen Inställningar har gjorts om helt för att vara mer logisk och täcka nästan alla behov, i ett försök att äntligen kunna pensionera den gamla Kontrollpanelen. Även om Kontrollpanelen fortfarande finns kvar för vissa djupa inställningar, är målet med Windows 11 att användaren aldrig ska behöva se den.

För eleverna och de spelintresserade har Windows 11 med sig några riktigt tunga nyheter under huven. Microsoft har hämtat teknik direkt från sina Xbox-konsoler för att maximera prestandan:

Auto HDR: En funktion som automatiskt förbättrar ljussättning och färger i äldre spel så att de ser modernare ut, förutsatt att du har en skärm som stöder det.

DirectStorage: Kanske den viktigaste tekniken för framtiden. Den tillåter grafikkortet att hämta data direkt från en snabb NVMe-disk utan att behöva gå den långa vägen via processorn. Detta innebär i praktiken att laddningstider i spel kan minskas från minuter till sekunder, och att spelvärldar kan bli betydligt mer detaljerade.

Trots den moderna ytan bygger Windows 11 fortfarande på den gamla stabila NT-kärnan. Om man gräver tillräckligt djupt, till exempel i fönstret för att hantera nätverkskort eller i diskhanteraren, hittar man fortfarande exakt samma gränssnitt som användes i Windows 2000 och XP. Det är en påminnelse om att oavsett hur mycket man målar om fasaden, så är fundamentet i Windows byggt för att vara kompatibelt med decennier av teknik.

6.1.8 Windows Server

Windows Server är i själva verket en "muskulös tvilling" till de versioner vi använder på våra vanliga arbetsdatorer. De delar exakt samma tekniska fundament – den stabila NT-kärnan. Det innebär att drivrutiner och grundläggande systemfiler ofta är identiska, och Microsoft släpper nästan alltid operativsystemen i par. Till exempel byggde Windows 7 på samma kod som Server 2008 R2, och dagens moderna Server 2022 delar arkitektur med Windows 10 och 11. Den stora skillnaden ligger inte i hur koden ser ut "under huven", utan i hur operativsystemet är optimerat och vilka tjänster det är redo att leverera.

Vid en första anblick ser skrivbordet i Windows Server bekant ut, men man märker snabbt att systemet är "rensat" från allt som hör hemma på en hemdator. Här finns inga förinstallerade spel, väder-appar eller underhållningstjänster. Istället möts administratören direkt vid inloggning av Server Manager. Detta fönster fungerar som serverns absoluta kommandocentral och nav. Det är härifrån man förvandlar en "tom" Windows-installation till en funktionell server genom att lägga till olika roller. Där en vanlig användare går till Microsoft Store för att hämta appar, går en systemadministratör till Server Manager för att installera de tjänster som hela nätverket är beroende av.

Det som verkligen definierar Windows Server är dess förmåga att agera som en central tjänsteleverantör för hundratals eller tusentals användare samtidigt. Genom att aktivera specifika roller kan servern ta på sig uppgifter som en vanlig Windows-dator helt saknar stöd för. Den viktigaste rollen är ofta Active Directory (AD), som fungerar som företagets digitala hjärna där alla användarkonton, lösenord och säkerhetsregler lagras centralt. Servern kan också hantera nätverkets infrastruktur genom roller som DNS (översätter webbadresser till IP-nummer) och DHCP (delar ut IP-adresser), eller fungera som en kraftfull webbserver via IIS (Internet Information Services).

Eftersom en server förväntas vara i drift dygnet runt, året om, är operativsystemet trimmat för maximal stabilitet och säkerhet. Det har stöd för betydligt kraftfullare hårdvara än en vanlig dator, såsom dussintals processorkärnor och flera terabyte RAM-minne. Denna enorma kapacitet och tillförlitlighet gör att Windows Server har en helt annan licensmodell och prislapp än konsumentversionerna. För de mest professionella miljöerna finns även alternativet Server Core, en version helt utan grafiskt gränssnitt där allt styrs via textkommandon. Det sparar både systemresurser och minskar attackytan för hackare, vilket gör servern till en så säker och effektiv arbetshäst som möjligt.

6.2 Windows uppbyggnad

Windows är ett modernt operativsystem med många komponenter som samarbetar för att ge en användarvänlig och flexibel plattform. Här går vi igenom några grundläggande byggstenar i Windows: enheter, filsystem, rättigheter (NTFS) och registret.

### 6.2.1 Enheter och enhetsbokstäver

I Windows representeras varje lagringsvolym som en enhet med en bokstav – C:, D:, E: och så vidare. Systemdisken är nästan alltid C: och innehåller själva Windows och programfiler. Historiskt är A: och B: reserverade för diskettenheter, vilket är varför C: blev standard för systemet. Även nätverksresurser kan “mappas” till enhetsbokstäver så att de beter sig som lokala enheter i Utforskaren.

När du ansluter en USB-disk, ett USB-minne eller sätter i en skiva i en optisk läsare får enheten normalt nästa lediga bokstav. Ordningen kan alltså variera mellan datorer eller tillfällen. Du kan ändra tilldelningen via Diskhantering (diskmgmt.msc) – praktiskt om du vill att externa enheter alltid ska ha samma bokstav eller om en applikation förväntar sig en viss sökväg. Var försiktig bara: program som installerats till en särskild enhetsbokstav kan sluta fungera om du ändrar bokstaven efteråt.

Det är också möjligt att montera en volym i en NTFS-mapp i stället för att ge den en bokstav (t.ex. C:\Data\Arkiv). Det kan vara en elegant lösning i servrar och labb där bokstäverna tar slut eller du vill gruppera lagring under en given katalog.

Sammanfattning

Windows identifierar lagringsenheter med enhetsbokstäver, t.ex. C:, D:, E:.

Systemdisken är oftast C: och innehåller Windows-installationen.

USB-minnen, externa hårddiskar och optiska enheter får automatiskt nästa lediga bokstav.

Du kan ändra enhetsbokstäver i Diskhantering (diskmgmt.msc).

### 6.2.2 Filsystem och NTFS

Ett filsystem bestämmer hur data organiseras på en volym: hur filer och mappar namnges, hur ledigt utrymme hanteras och hur integritet och rättigheter upprätthålls. I Windows är NTFS standard på systemdisken. Det ersatte äldre FAT32 eftersom NTFS skalar bättre till stora diskar och ger fler funktioner för pålitlighet och säkerhet.

NTFS har journalföring som minskar risken för korruption vid strömavbrott, åtkomsträttigheter (ACL) på fil- och mappnivå, komprimering per mapp/fil och stöd för mycket stora filer och volymer. Det finns också avancerade egenskaper som kryptering (EFS), diskkvoter, hårda länkar/symboliska länkar och alternativa dataströmmar. Tillsammans gör detta NTFS lämpligt för både klienter och servrar, medan FAT32 eller exFAT främst används för flyttbara medier och kompatibilitet mellan olika system.

I praktiken formateras system- och datavolymer i Windows nästan alltid med NTFS. För externa enheter som behöver fungera mellan många olika operativsystem kan exFAT vara ett kompromissval – det saknar NTFS:s rättigheter och journaling men har bättre kompatibilitet än NTFS och färre begränsningar än FAT32.

Sammanfattning

Det vanligaste filsystemet i Windows är NTFS (New Technology File System).

NTFS jämfört med FAT32:

Stöd för stora filer och diskar.

Möjlighet till filkomprimering.

Journalföring (skydd mot datakorruption).

Avancerade rättigheter och säkerhetsinställningar.

Alla moderna Windows-versioner (inkl. Server) använder NTFS som standard på systemdisken.

### 6.2.3 Rättigheter och behörigheter

NTFS hanterar åtkomst via åtkomstkontrollistor (ACL) där varje fil eller mapp har poster (ACE) som beskriver vilka användare eller grupper som får göra vad. Rättigheter kan sättas direkt på en mapp eller fil, men ärvs normalt nedåt i strukturen för att underlätta hanteringen. Ett typiskt upplägg är att ge rättigheter till grupper i stället för enskilda personer, och låta gruppmedlemskap styra åtkomst.

Vanliga rättigheter är Läs, Skriv, Ändra och Fullständig behörighet. Ägaren av en fil eller mapp kan alltid ändra dess rättigheter; Administratörer kan dessutom ta ägarskap vid behov. I Utforskaren hittar du detta under Egenskaper → Säkerhet, och i Avancerat ser du arv, ärvda/explicita poster och kan räkna ut effektiva behörigheter.

I nätverksmiljöer spelar även delningsrättigheter in (fliken Delning). Den faktiska åtkomsten över nätverket blir kombinationen av delningsrättigheter och NTFS-rättigheter – i praktiken gäller den mest restriktiva av de två. En robust princip är att hålla delningen relativt generös (t.ex. Läs) och finjustera åtkomst med NTFS på mappnivå.

Sammanfattning

NTFS-rättigheter styr vem som får göra vad med filer och mappar.

Du kan ange rättigheter via Egenskaper → Säkerhet i Utforskaren.

Exempel på rättigheter: Läs, Skriv, Ändra, Fullständig behörighet.

Rättigheter kan ärvas från mappstrukturen – bra för att hantera stora filsystem.

Administratörer kan alltid ta ägarskap och ändra behörigheter vid behov.

Viktigt för säkerhet i både hem- och företagsmiljöer.

### 6.2.4 Registret

Windows-registret är en hierarkisk databas som lagrar konfiguration för operativsystemet, drivrutiner, tjänster och många applikationer. Strukturen består av nycklar (som mappar) och värden (som filer) med olika datatyper, till exempel REG_SZ (text) eller REG_DWORD (heltal). Rotnivån utgörs av så kallade hives: HKEY_LOCAL_MACHINE (HKLM) – maskinvida inställningar för OS, drivrutiner och tjänster; HKEY_CURRENT_USER (HKCU) – inställningar för den inloggade användaren; HKEY_USERS (HKU) – alla användarprofiler (används bl.a. för att nå andra användares HKCU); HKEY_CLASSES_ROOT (HKCR) – filtypsassociationer och COM-registreringar; samt HKEY_CURRENT_CONFIG (HKCC) – en ögonblicksbild av aktuell hårdvaruprofil och vissa drivrutinsinställningar.

Registret kan redigeras med Registereditorn (regedit). Det är ett kraftfullt verktyg för felsökning och avancerad konfiguration när det saknas GUI-inställningar, och det används också av gruppolicyer och många installationsprogram. Det går att exportera delar av registret till .reg-filer för att spara och återställa inställningar.

Samtidigt är registret en känslig del av systemet. Felaktiga ändringar kan orsaka instabilitet eller göra Windows obrukbart. En säker arbetsmetod är att först skapa en systemåterställningspunkt, exportera de nycklar du tänker ändra och dokumentera exakt vad som ändrats, så att du kan rulla tillbaka vid behov.

Windows-registret är en hierarkisk databas som lagrar inställningar för operativsystemet och installerade program.

Består av nycklar och värden – ungefär som mappar och filer.

Centralt för konfiguration av Windows, drivrutiner och applikationer.

Kan redigeras med Registereditorn (regedit) – kraftfullt men riskabelt.

Viktigt att vara försiktig: felaktiga ändringar kan göra systemet instabilt eller obrukbart.

Windows bygger på en kombination av tydliga enhetsbokstäver, avancerat filsystem med säkerhetsfunktioner (NTFS), och ett centralt register som styr konfigurationen. Att förstå dessa delar är viktigt för både grundläggande användning och mer avancerad systemadministration.

### 6.3.2 Fördelar med Windows

Windows har under decennier varit det dominerande operativsystemet på marknaden, och det finns flera tunga skäl till att både privatpersoner och gigantiska företag fortsätter att välja plattformen. Det handlar om en kombination av igenkänning, ett enormt utbud av verktyg och en teknisk infrastruktur som är byggd för att skala uppåt.

6.3.2.1 Användarvänligt gränssnitt

Den kanske mest uppenbara fördelen med Windows är dess välbekanta gränssnitt. Sedan Windows 95 har grundpelarna – Start-menyn, aktivitetsfältet och det grafiska skrivbordet – sett ungefär likadana ut. Denna konsekvens gör att tröskeln för att lära sig systemet är mycket låg. Även en nybörjare kan snabbt förstå hur man navigerar mellan mappar, startar program och ändrar grundläggande inställningar utan att någonsin behöva röra en kommandorad.

För en IT-tekniker innebär detta också att det är lättare att ge support till slutanvändare. Eftersom de flesta redan är vana vid Windows logik, behöver man sällan utbilda personalen i hur man utför de enklaste sysslorna. Gränssnittet är designat för att vara intuitivt, vilket sparar både tid och frustration i en arbetsmiljö.

6.3.2.2 Brett programstöd

Den största tekniska styrkan hos Windows är utan tvekan det enorma utbudet av mjukvara. Eftersom Windows har den största marknadsandelen, väljer nästan alla mjukvaruleverantörer att utveckla sina program för Windows i första hand. Det spelar ingen roll om det handlar om avancerade grafikprogram som Adobe Creative Cloud, bokföringssystem eller specialiserade ingenjörsverktyg – chansen är nästan hundraprocentig att de fungerar felfritt på Windows.

Inom spelvärlden är Windows i princip ohotad etta tack vare tekniker som DirectX, vilket gör att nästan alla datorspel utvecklas med Windows som huvudplattform. För ett företag innebär detta programstöd en trygghet; man vet att man inte kommer att bli låst från viktiga verktyg på grund av operativsystemet, vilket ofta kan vara fallet med mer nischade alternativ.

6.3.2.3 Hårdvarukompatibilitet

Windows är byggt för att fungera på miljontals olika hårdvarukombinationer. Oavsett om du bygger en egen dator av delar från tio olika tillverkare eller köper en färdig märkesdator, så "bara fungerar" det oftast. Detta beror på att hårdvarutillverkare prioriterar att skriva drivrutiner för Windows före allt annat. Tekniker som Plug and Play har förfinats under tre decennier, vilket gör att installation av skrivare, grafikkort och nätverksutrustning oftast sker helt automatiskt.

Till skillnad från vissa Linux-distributioner, där man ibland kan behöva kompilera egna drivrutiner eller leta efter speciallösningar för att få ett trådlöst nätverkskort att fungera, hanterar Windows detta i bakgrunden. För en IT-avdelning som hanterar hundratals olika datormodeller är denna breda kompatibilitet en enorm fördel som minskar mängden manuellt arbete vid driftsättning.

6.3.2.4 Backat av Microsoft

Att ha ett av världens största teknikföretag i ryggen ger en stabilitet som är svår att matcha. Microsoft garanterar långsiktig support och regelbundna säkerhetsuppdateringar, vilket är kritiskt för verksamheter som inte har råd med driftstopp. Genom Microsoft 365-integrationen blir Windows också en del av ett större ekosystem där OneDrive, Teams och Outlook fungerar sömlöst tillsammans, vilket skapar en effektiv digital arbetsplats.

Denna support innebär också att det finns en enorm mängd dokumentation, certifieringar och färdiga lösningar för i princip alla problem som kan uppstå. Som tekniker är du aldrig ensam; om något går fel finns det nästan alltid en officiell patch eller en välkänd lösning tillgänglig, vilket ger en trygghet som "community-drivna" projekt ibland kan sakna i affärskritiska situationer.

6.3.2.5 Professionella verktyg och serverfunktioner

Windows styrka ligger inte bara i vad den enskilda användaren ser, utan i hur systemet kan hanteras centralt. Genom verktyg som Active Directory (AD) och Group Policy (GPO) kan en administratör styra tusentals datorer från en enda server. Man kan bestämma exakt vilka program som får installeras, hur skrivbordet ska se ut och vilka säkerhetsregler som ska gälla för hela företaget med några få klick.

Denna skalbarhet gör att Windows kan växa med användaren. Samma grundläggande kunskaper man lär sig på en vanlig laptop kan appliceras på kraftfulla servrar med Hyper-V för virtualisering eller avancerade databashanterare. Det gör Windows till en komplett plattform som sträcker sig hela vägen från barnets första speldator till hjärtat i ett globalt storbolags serverhall.

### 6.3.3 Nackdelar

Trots sin dominans och användarvänlighet är Windows långt ifrån ett perfekt system. För en IT-avdelning innebär valet av Windows en ständig balansgång mellan funktionalitet och de begränsningar som följer med en kommersiell, stängd plattform. Från höga kostnader till resurskrävande mjukvara finns det flera aspekter som kan tala emot Windows i vissa specifika miljöer.

#### 6.3.3.1 Licenskostnad

Den mest påtagliga nackdelen med Windows, särskilt i jämförelse med Linux, är att det inte är gratis. För en privatperson är kostnaden ofta inbakad i priset på datorn, men för ett företag är licenshantering en betydande budgetpost. Varje enskild klientdator kräver en licens, och när det kommer till Windows Server blir kostnaderna snabbt mycket höga.

Inom servervärlden räcker det dessutom sällan med en grundlicens; man behöver ofta köpa till så kallade CALs (Client Access Licenses) för varje användare eller enhet som ansluter till servern. För en organisation med hundratals anställda innebär detta en komplex och dyr administration som måste förnyas regelbundet för att man ska vara lagligt licensierad.

#### 6.3.3.2 Stängd källkod

Windows är ett "proprietärt" operativsystem, vilket innebär att källkoden är stängd och ägs helt av Microsoft. Som användare eller tekniker har du ingen möjlighet att se exakt hur systemet fungerar under huven, och du får absolut inte modifiera koden för att anpassa den efter dina behov. Detta skapar ett totalt beroende av Microsofts utvecklingsavdelning.

Om Microsoft bestämmer sig för att ändra ett gränssnitt (som i Windows 8) eller lägga ner en funktion som ditt företag är beroende av, har du inget annat val än att anpassa dig. Du kan inte heller själv laga säkerhetshål eller buggar i systemet, utan måste vänta på att Microsoft prioriterar och släpper en officiell uppdatering. Detta "inlåsningseffekt" (vendor lock-in) är en av de främsta anledningarna till att vissa organisationer föredrar öppen källkod.

#### 6.3.3.3 Resurskrävande

Windows har ett rykte om att vara ett "tungt" operativsystem. Eftersom Microsoft strävar efter att behålla kompatibilitet med gammal mjukvara, bär systemet på en enorm mängd gammal kod som tar upp plats och kraft. Varje ny version av Windows tenderar att kräva mer arbetsminne (RAM), snabbare processorer och mer diskutrymme än den föregående.

Detta märks tydligt i och med Windows 11, där kraven på moderna processorer och TPM-chipp har gjort att miljontals fullt fungerande datorer plötsligt anses föråldrade. För en IT-avdelning innebär detta att man tvingas till dyra hårdvaruuppgraderingar långt innan den fysiska utrustningen faktiskt har gått sönder, bara för att kunna köra den senaste och säkraste mjukvaran.

#### 6.3.3.4 Ett primärt säkerhetsmål

På grund av sin enorma marknadsandel är Windows den absolut största måltavlan för cyberkriminella. Om man skriver ett virus eller ett ransomware-program vill man nå så många offer som möjligt, och då är Windows det självklara valet. Detta innebär att en Windows-dator i princip aldrig kan lämnas oskyddad; den kräver aktivt underhåll i form av ständiga säkerhetsuppdateringar och avancerade antivirusprogram (EDR/XDR).

Som tekniker innebär detta en ständig kamp mot klockan. Varje månad släpper Microsoft dussintals patchar på den så kallade "Patch Tuesday", och dessa måste testas och rullas ut snabbt för att täppa till sårbarheter. Denna osäkerhet är en inbyggd kostnad i Windows-ekosystemet – man köper ett system som är lätt att använda, men man betalar priset i form av ett konstant behov av övervakning och skydd.

#### 6.3.3.5 Komplexitet i företagsmiljöer

Även om Windows är enkelt för slutanvändaren, kan administrationen av ett stort Windows-nätverk vara extremt komplex. Att hålla reda på olika versioner (Home, Pro, Enterprise), olika licensmodeller (OEM, Volume Licensing, Subscription) och se till att alla följer företagets policyer kräver djup expertkunskap.

Utan rätt verktyg och utbildning blir det snabbt övermäktigt att hantera tusentals datorer centralt. Dessutom kan de automatiska uppdateringarna, som är tänkta att hjälpa till, ibland orsaka stora problem om en uppdatering visar sig vara inkompatibel med ett affärskritiskt program. Att navigera i denna "licensdjungel" och tekniska komplexitet är ett heltidsjobb för många systemadministratörer, vilket är en dold nackdel som ofta glöms bort vid ett inköp.

### 6.3.4 Sammanfattning: Att välja rätt system för rätt behov

Sammanfattningsvis är Windows ett kraftfullt, välkänt och flexibelt operativsystem som erbjuder ett oöverträffat stöd för både mjukvara och hårdvara. Dess dominans på marknaden gör det till det självklara valet för de flesta företag och hemanvändare som kräver total kompatibilitet och ett gränssnitt som nästan alla redan behärskar. Från de minsta bärbara datorerna till de största serverhallarna utgör Windows den digitala ryggraden i stora delar av vår värld.

Samtidigt innebär valet av Windows att man måste vara beredd att hantera en del specifika utmaningar. Som vi har sett är det inte ett kostnadsfritt system; licenskostnader, krav på modern och kraftfull hårdvara samt ett ständigt behov av aktivt säkerhetsarbete är faktorer som en IT-tekniker alltid måste kalkylera med. Windows kräver mer underhåll i form av uppdateringar och virusskydd än många av sina konkurrenter.

Att förstå dessa styrkor och svagheter är helt avgörande för att du som tekniker ska kunna välja rätt system för rätt behov. Det finns situationer där Windows är det enda logiska valet på grund av dess professionella verktyg och breda stöd, men det finns också miljöer där nackdelarna väger tyngre. Genom att ha med dig denna helhetsbild kan du fatta välgrundade beslut, oavsett om det gäller en enkel dator för privat användning eller komplexa, storskaliga IT-lösningar för globala organisationer.

Hantering av grupper och konton:

Inställningar → Konton → Andra användare – för enkel hantering.

Lokal användar- och grupphantering (lusrmgr.msc) – i Pro-versioner för att mer avancerat skapa, ändra eller ta bort konton och grupper.

Viktigt att förstå: Grupper används för att enkelt tilldela rättigheter till flera användare utan att behöva sätta rättigheter individuellt på varje användare.

Rättigheter i filsystemet (NTFS)

Windows använder NTFS (New Technology File System) för att hantera rättigheter på filer och mappar. Det låter dig styra exakt vad olika användare eller grupper får göra.

Exempel på rättigheter:

Läsa: Öppna filer, visa innehåll.

Skriva: Skapa eller ändra filer.

Ändra: Kombinerar läsa, skriva och ta bort.

Fullständig behörighet: Alla ovanstående + ändra ägare och rättigheter.

Ärvda rättigheter

Som standard ärver nya filer och mappar rättigheter från den mapp de ligger i.

Det gör det enklare att hantera behörighet i hela mappstrukturer.

Du kan bryta ärvning om du behöver anpassa rättigheterna för en specifik fil eller mapp.

Hantera rättigheter

Högerklicka på fil eller mapp → Egenskaper → Säkerhet.

Här kan du se och ändra vilka användare och grupper som har rättigheter.

Under Avancerat kan du se ärvda rättigheter och ändra ägarskap.

Viktigt att veta:

Administratörer kan alltid ta ägarskap över filer eller mappar, även om de saknar andra rättigheter.

Att förstå och använda rättigheter korrekt är en viktig del av säkerheten både hemma och i företagsmiljö.

6.6 Installation av Windows Server i fysisk maskin/VM

Att installera Windows Server skiljer sig inte mycket från installationen av Windows 11, men det finns några viktiga skillnader att känna till. Framför allt finns det olika editioner och möjligheten att installera med eller utan grafiskt gränssnitt (GUI).

I denna instruktion utgår vi ifrån att du redan har en ISO-fil tillgänglig.

### 6.6.1 Skapa installationsmedia med Rufus

För att installera Windows Server på en fysisk dator behöver du ett bootbart USB-minne.

Ladda ner och starta Rufus (https://rufus.ie/)

Välj din USB-enhet under Device

Klicka på SELECT och välj ISO-filen för Windows Server

Partition scheme: MBR (för äldre BIOS) eller GPT (för UEFI)

Klicka på START

När klart – starta om datorn och ändra bootordning i BIOS/UEFI

### 6.6.2 Installera i en virtuell miljö

Om du använder t.ex. Proxmox, VirtualBox eller Hyper-V:

Skapa en ny virtuell maskin

Tilldela RAM (minst 4 GB) och lagringsutrymme (minst 40 GB)

Montera ISO-filen som virtuell CD/DVD

Starta upp och följ installationsguiden

6.6.3 Editioner: Standard, Datacenter och Core/Desktop Experience

Vid installation kommer du att få välja mellan olika versioner. Här är en kort förklaring:

### 6.6.4 Steg-för-steg-installation

När ISO är startad:

Välj rätt edition (t.ex. "Standard Desktop Experience")

Acceptera licensvillkor

Välj Custom installation

Skapa/markera en partition → Next

Vänta medan Windows installeras

Skapa administratörslösenord

Ge servern ett namn

Ställa in IP-adress

Köra Windows Update

Eventuellt lägga till roller/funktioner via Server Manager
