# 3.4 Containrar – En introduktion

Containrar är ett modernt och effektivt sätt att isolera applikationer utan att behöva köra ett helt operativsystem för varje program. Till skillnad från virtuella maskiner, som emulerar hela hårdvaran, delar containrar på själva kärnan (kernel) i värdens operativsystem. Det gör dem otroligt snabba och resurssnåla.

Ett klassiskt problem för utvecklare är meningen: "Men det fungerade ju på min dator!". Med containrar, som Docker eller Podman, kan man paketera programmet tillsammans med exakt alla filer och inställningar som behövs. På så sätt är man garanterad att det fungerar likadant oavsett om det körs på din laptop eller på en server i molnet.

Ett exempel skulle kunna vara att du har ett behov av att kunna köra Adobe Photoshop. I en Windows- eller Mac-miljö är det inga problem, men på ett Linuxsystem fungerar det inte. För att lösa problemet så skapar du en container med programmet, och de nödvändiga delarna från Windows som behövs för att köra det. Nu kan du köra det i valfri miljö utan problem.

Även om containrar teoretiskt skulle kunna användas för grafiska program som Photoshop, används de i praktiken nästan uteslutande för serverapplikationer och mikrotjänster. Det beror på att containrar är designade för att vara "statlösa" och flyttbara, medan tunga grafiska program kräver djup integration med grafikkort och specifika drivrutiner som containrar inte är byggda för att hantera.

Exempel på containerplattformar:

Docker

Podman

Kubernetes (för att hantera många containrar)

Containrar används ofta tillsammans med virtualisering eller i molnmiljöer. Det är bra att känna till skillnaden:

Virtuella maskiner emulerar hela hårdvaran och kör ett eget OS.

Containrar delar värd-OS men isolerar applikationen.
