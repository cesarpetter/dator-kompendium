# 2.6 Bootloaders

En bootloader är ett litet men kritiskt program som laddas in av datorns firmware – BIOS eller UEFI – direkt efter att den första systemkontrollen (POST) har genomförts. Bootloaderns uppgift är att initiera hårddisken, lokalisera och ladda in operativsystemets kärna (kernel) i RAM-minnet och därefter lämna över kontrollen till operativsystemet. Det är alltså bootloadern som är länken mellan hårdvaran och själva operativsystemet, och utan den skulle systemet aldrig kunna startas upp. Exempel på bootloaders inkluderar GRUB (Linux), Windows Boot Manager, och systemd-boot.

Exempel på bootloaders är:

Windows Boot Manager (BCD-store)

GRUB (Grand Unified Bootloader) för Linux

boot.efi för macOS

Dessa lagras på olika sätt beroende på vilken firmware-standard som används:

BIOS: MBR (512 byte bootstrap + ~31 KB VBR) pekar vidare

UEFI: EFI-systempartition (ESP) innehåller .efi-filer

### 2.6.1 Exempel på uppstartssekvenser

För att underlätta felsökning är det bra att veta exakt vilka steg de vanligaste operativsystemen tar under uppstarten. Om datorn stannar mellan två steg kan man ofta räkna ut vilken komponent eller fil som är skadad.

Windows (UEFI)

Ström på → Nätaggregatet skickar Power Good.

UEFI POST → Hårdvaran kontrolleras.

ESP läses → UEFI letar upp EFI-systempartitionen på disken.

Laddar bootmgfw.efi → Windows huvudladdare startas.

Läser BCD → Boot Configuration Data (menyn/inställningarna) läses in.

Laddar winload.efi → Operativsystemets kärna (Kernel) läses in i RAM.

Kernel startar → Windows-logotypen visas och drivrutiner laddas.

Linux (GRUB via UEFI)

Ström på → Firmware vaknar.

UEFI POST → Hårdvarukontroll.

Laddar grubx64.efi → UEFI hämtar GRUB-laddaren från ESP-partitionen.

GRUB-meny → Användaren väljer operativsystem (eller så väljs standardalternativet).

Laddar Kernel & initramfs → GRUB läser in vmlinuz och de drivrutiner som behövs för att starta.

Kernel startar → Systemet påbörjar uppstarten av tjänster.

### 2.6.2 Vanliga problem vid uppstart

Här är några av de vanligaste felen man stöter på:

"No bootable device found": Datorn hittar ingen enhet med en giltig bootloader. Detta kan bero på fel startordning eller att hårddisken är trasig/lossnat.

Secure Boot-blockering: UEFI vägrar starta en bootloader som inte har en godkänd digital signatur (vanligt vid installation av vissa Linux-distros).

MBR/GPT-konflikter: En dator inställd på "Legacy BIOS" kan inte starta en disk formaterad med det moderna GPT-schemat, och vice versa.

Korrupt bootloader: Om winload.efi eller grub.cfg saknas kommer uppstarten att avbrytas med ett felmeddelande efter att firmware har gjort sitt jobb.
